## Decisions Made

- **Self-contained fixtures (vs. reuse of `two_tenants` from conftest.py)**: the existing `two_tenants` fixture seeds Category with legacy column shape (no `code` / `ord` NOT NULL) and currently raises `NotNullViolationError` on a v1.0 schema. Fixing it is out of scope for this plan; per CLAUDE.md / SCOPE BOUNDARY rules, the deviation is recorded but not patched here. Tests in this module use their own `_seed_*` helpers and `_truncate_v1_tables`.
- **Two-session architecture**: conftest.py promotes `ADMIN_DATABASE_URL` to `DATABASE_URL` for general test convenience. The original runtime URL is preserved as `RUNTIME_DATABASE_URL`. We open a second session bound to that URL inside the `runtime_session` fixture for any test asserting RLS enforcement; admin (`db_session`) is used only for seeding and verification reads.
- **Composite FK tests use `db_session`** (admin) — FK violations are enforced regardless of role, so we keep those simpler. The composite FK alone is enough to assert cross-tenant blockage; running them under runtime would add no signal.
