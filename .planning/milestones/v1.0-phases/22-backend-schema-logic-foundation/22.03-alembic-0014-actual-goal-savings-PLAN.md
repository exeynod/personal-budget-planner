| pg enum →app | actualkind enum has 4 values; PlannedTransaction app-layer validators must reject roundup/deposit |
| db→app via parent_txn_id self-FK | parent_txn_id ON DELETE CASCADE means deleting parent removes roundup child automatically |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-03-01 | Tampering | Planned txn with kind='roundup' | mitigate | App-layer Pydantic validator on PlannedTransactionCreate rejects kind ∉ {expense, income}; documented in plan 22.5 model + plan 22.12 schemas |
| T-22-03-02 | Information Disclosure | actual_transaction.parent_txn_id RLS gap | mitigate | RLS policy on actual_transaction (existing from 0006) covers child rows; composite FK in plan 22.4 prevents cross-tenant linkage at DB level |
| T-22-03-03 | Denial of Service | autocommit_block partial failure | mitigate | `ADD VALUE IF NOT EXISTS` is idempotent; rename happens first — if rename succeeds but value-add fails, retry succeeds; document recovery in docstring |
| T-22-03-04 | Tampering | day_of_month value > 28 | mitigate | CHECK constraint `day_of_month BETWEEN 1 AND 28` enforces at DB; covers all months including February |
| T-22-03-05 | Tampering | savings_config.roundup_base bypass | mitigate | CHECK constraint `roundup_base IN (10, 50, 100)` enforces at DB |
| T-22-03-06 | Repudiation | rollover_processed_at race | mitigate | Partial unique index + advisory lock (plan 22.10) ensures rollover runs at most once per period |
| T-22-03-07 | Spoofing | savings_config.user_id PK | mitigate | PK = user_id with FK ON DELETE CASCADE; no user can have two configs; revoke flow purges automatically |
