## Deviations from Plan

### [Rule 3 - Scope correction] Did NOT add `account_id` and `occurred_at` to ActualTransaction

- **Found during:** Task 1
- **Issue:** The user-prompt scope §4 lists adding `account_id` and `occurred_at` to ActualTransaction. However, alembic migration 0014 (the truth source) does NOT add either column to `actual_transaction` — it only adds `parent_txn_id`. `account_id` is added only to `subscription`. The user-prompt also hedges with "if not present" for `occurred_at`.
- **Fix:** Followed migrations as truth source per user instruction "Migrations 0012-0015 all at HEAD; the dev DB schema reflects all 4 migrations. This plan updates `app/db/models.py` so the ORM mirrors the live schema." Did not invent columns the schema doesn't have.
- **Files modified:** none — decision-only deviation
- **Verification:** Cross-checked all `op.add_column("actual_transaction", ...)` calls in 0014 — confirmed `parent_txn_id` is the sole addition.
- **Note for downstream:** If service-layer plans 22.06+ require `actual_transaction.account_id` (e.g., for roundup hook to inherit `parent.account_id`), it must be added in a NEW migration (e.g., `0016_v10_actual_account_id.py`) plus an ORM follow-up. CONTEXT §Area 3 D-03 mentions `account_id=parent.account_id` in roundup, so this gap is real. **Recommended:** add to backlog for plan 22.06 (roundup service) — it will need `account_id` either via direct lookup or by joining `parent_txn.account_id`. Currently DATA-MODEL.md §1.4 specifies `Transaction.accountId` as a field, but the schema doesn't reflect it yet.

### [Rule 2 - Missing critical] Mirrored DB-level CHECK/UNIQUE constraints in ORM `__table_args__`

- **Found during:** Task 1
- **Issue:** PLAN.md task action describes adding columns but doesn't explicitly call out adding CHECK constraints (rollover-enum, ord-format, balance-range, day-of-month, target-positive, name-length, savings-base) and composite UNIQUE constraints to ORM `__table_args__`. Without them, alembic autogenerate would re-propose them as new migrations, and they wouldn't appear in `MetaData` for test fixtures.
- **Fix:** Added all CHECK and UNIQUE constraints from migrations 0012-0015 to corresponding `__table_args__` tuples (Category, Account, Subscription, Goal, SavingsConfig, BudgetPeriod, ActualTransaction). The constraints already exist on DB; ORM mirrors them for metadata alignment.
- **Files modified:** `app/db/models.py` (only)
- **Verification:** `python -c "import ast; ast.parse(...)"` exits 0; `configure_mappers()` exits 0 inside api container.
- **Committed in:** `95a884d`
