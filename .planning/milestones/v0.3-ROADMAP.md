## Overview

Функциональный редизайн nav (5 табов), новый экран Аналитики с трендами и прогнозом, conversational AI-помощник с tool-use над данными бюджета, AI-категоризация в форме новой транзакции через эмбеддинги. ### Phase 10: AI Categorization
**Goal**: AI-предложение категории в форме «Новая транзакция» через embeddings и cosine similarity, без LLM-вызова на каждой транзакции (только embedding API)
**Depends on**: Phase 9 (LLM-клиент готов)
**Requirements**: AICAT-01..06, SET-03
**Success Criteria** (achieved):
  1. При вводе описания в `ActualEditor` через ~500ms debounce backend возвращает топ-1 категорию по cosine similarity description-embedding и cached category-embeddings
  2. Если confidence ≥ 0.5 — UI показывает AI-suggestion box (имя + confidence-bar) вместо `<select>`; кнопка «Сменить» возвращает обычный select
  3. Если confidence < 0.5 — UI показывает обычный select без AI-навязывания
  4. Эмбеддинги категорий хранятся в новой таблице `category_embedding(category_id PK, vector(1536), updated_at)`; pgvector extension добавлен в Postgres init; перегенерация при изменении имени категории
  5. Toggle `enable_ai_categorization` в Settings (`SET-03`) отключает feature; default = on
  6. Pgvector index (HNSW) для быстрого cosine search

## Milestone Summary

**Decimal Phases (INSERTED):**
- Phase 10.1: AI Cost Optimization — inserted after Phase 10 для pre-ПСИ cost-review (~50% input-токен reduction)
- Phase 10.2: AI Hardening + Write-Flow — inserted после live UAT, который нашёл 6 latent багов (главный: `OPENAI_API_KEY` никогда не пробрасывался в api контейнер)

**Key Decisions:**
- Bottom nav реорганизуется в функциональную: Главная / Транзакции / Аналитика / AI / Управление. Заменяет MVP-nav (История/План/Подписки/Ещё) — История+План объединяются в «Транзакции» с под-табами, Подписки+Шаблон+Категории+Настройки в «Управление»
- LLM provider = OpenAI (gpt-4.1-nano для chat, text-embedding-3-small для эмбеддингов); абстрактный LLM-клиент с `LLM_PROVIDER` ENV для возможности переключения
- AI Categorization (Phase 10) через embeddings + cosine similarity, без LLM-вызова на каждой транзакции — pgvector в Postgres
- Аналитика повышена из v2 в v1 — агрегаты считаются на backend, SVG-чарты самописные без chart-libs
- AI write-flow: propose-and-approve (Phase 10.2) — AI **никогда не пишет в БД молча**; всегда открывает prefilled bottom-sheet
- Synonym-augmented embeddings (Phase 10.2) — embedding source = `name + synonyms` для лучшего matching коротких русских описаний
- gpt-4.1-nano → gpt-4.1-mini upgrade (Phase 10.2) — nano выдумывал категории и плохо считал
- English system-prompt + tool schemas (Phase 10.1) — ~2.3× token compaction на Cyrillic vs Latin
- pgvector index = HNSW (Phase 10) — для 14 категорий любой ок, HNSW проще
