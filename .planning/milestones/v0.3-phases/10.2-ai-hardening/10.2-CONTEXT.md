# Phase 10.2: AI Hardening + Write-Flow (INSERTED)

**Inserted after:** Phase 10.1
**Reason:** Pre-ПСИ live UAT-сессия выявила, что `OPENAI_API_KEY` ни
разу не пробрасывался в api-контейнер с момента Phase 9 — это значит,
что весь AI-surface (chat tool-calls, embedding refresh, /ai/suggest-
category, embedding-on-create) **никогда не выполнялся в живой среде**
до этой сессии. Тесты с моками не валидировали ни структуру payload'ов
к OpenAI, ни pgvector cast, ни AsyncSession-commit-семантику. Как
только ключ заработал — посыпались latent баги Phase 9/10. Параллельно
пользователь запросил write-flow (AI добавляет транзакции/план через
чат) и upgrade модели для лучшей аналитики.

## Goal

Сделать AI-surface действительно работающим в проде: закрыть все
latent баги, пересобрать embedding-pipeline под реалии
text-embedding-3-small, добавить proposal-flow для записи через чат,
переехать на gpt-4.1-mini.

## Scope

### A. Infra wiring (предусловие для всего остального)
1. Проброс `OPENAI_API_KEY` + `LLM_PROVIDER` / `LLM_MODEL` /
   `AI_MAX_CONTEXT_MESSAGES` / `ENABLE_AI_CATEGORIZATION` в
   docker-compose.yml (только api-сервис) и .env.example.
2. Fail-fast валидатор в `Settings.validate_production_settings()` —
   api отказывается стартовать без валидного ключа независимо от
   `DEV_MODE`. SERVICE-aware: bot/worker не требуют ключа.
3. Тестовая инфра: фейк-токен в `tests/conftest.py` и
   `docker-compose.test.yml` для интеграционных тестов.

### B. Pre-existing AI-pipeline баги (раскрылись из-за A)
4. Embedding background-task падал с FK violation — нужен явный
   `db.commit()` ДО `background_tasks.add_task(...)`.
5. `_refresh_embedding` async-with НЕ авто-коммитит — INSERT
   откатывался; добавлен явный `await session.commit()`.
6. `EmbeddingService.suggest_category` посылал Python `list` в
   pgvector cast — `asyncpg.DataError`. Сериализация через строковый
   литерал `'[0.1,0.2,...]'`.
7. `get_embedding_service()` создавал новый instance на каждый
   запрос → LRU кэш на throw-away service никогда не hitнул.
   `@lru_cache(maxsize=1)` → singleton.
8. Tool round-trip: перед `{role: tool, ...}` теперь добавляется
   `{role: assistant, content: null, tool_calls: [...]}` — иначе
   OpenAI кидает 400 «messages with role 'tool' must be a response
   to a preceeding message with 'tool_calls'».
9. `Decimal` из `SUM(BIGINT)` ломал `json.dumps` — приведение
   `int(... or 0)` на всех агрегатах + защитный `default=str`.
10. История разговора реконструируется с синтетическими
    tool_call_id'ами (`reconstructed_N`), чтобы LLM на повторных
    turn'ах помнил факты из ранее вызванных tool'ов.
11. Agent loop (max_rounds=5) — модель может вызывать tools
    последовательно и параллельно в одном чат-turn'е; раньше путь
    был линейным «один tool → один LLM-ответ».

### C. UX-фиксы AI-чата
12. Empty-bubble и рассинхрон контекста: `/ai/history` отдаёт только
    user/assistant с непустым content; в LLM-context разворачиваются
    в правильную OpenAI-структуру.
13. SYSTEM_PROMPT: запрет упоминать имена функций / tool / JSON;
    инструкция включать current balance + daily expense в любой
    forecast-ответ; запрет шортхенда «150 000 ₽ → 15 тыс».
14. Chat input: auto-grow textarea с потолком `40dvh`, scroll внутри
    после превышения.
15. AI-suggestion box в ActualEditor: убран confidence-bar
    (бесполезен после фильтра порогом), кнопка «Сменить» сделана
    видимым интерактивным элементом.

### D. Embedding tuning (recall на коротких терминах)
16. `SUGGEST_THRESHOLD` 0.5 → 0.35 — на text-embedding-3-small
    короткие русские термины редко берут 0.5.
17. Synonym-augmented embeddings: каждая seed-категория embed-ится
    как `"Имя: синоним1, синоним2, …"`. Display name не меняется.
    «Пятёрочка»→Продукты теперь 0.36, «бензин»→Транспорт 0.37,
    «Хоккей»→Спорт 0.37 — все выше threshold; gibberish
    (`qwertyuiop`=0.18) по-прежнему отсекается.

### E. AI factual-analyst tone
18. SYSTEM_PROMPT: «You are an analyst, not an advisor. Present facts
    and let the user decide. NEVER tell the user to cut a category
    just because it's high — you don't know which expenses are
    essential ('Здоровье' may be vital medication).» Routing rules
    1..3 переписаны под факто-ориентированные ответы (для «где
    экономить» — только перерасход факт > план, без советов резать).

### F. Write-flow через чат (новая фича)
19. Два новых tool'а — `propose_actual_transaction` и
    `propose_planned_transaction`. Они НЕ пишут в БД, а возвращают
    `{_proposal: true, txn: {...}}` с резолвом категории через
    embedding (`description` only — сервер сам, без `category_hint`
    от LLM, иначе nano выдумывал «Здоровье» для «кофе на работе»).
20. Route эмитит SSE-event `{type: "propose", data: {...}}` сверх
    обычного `tool_end`, тот же payload идёт обратно в LLM как
    tool-response.
21. Frontend: `AiStreamEvent` стал discriminated union с типизацией
    Proposal payload. `useAiConversation` ловит `propose` →
    `proposal` state. Новый компонент `AiProposalSheet` рендерит
    `BottomSheet` с `ActualEditor` или `PlanItemEditor` с
    pre-filled полями. Save идёт через стандартные REST endpoints —
    AI **никогда** не пишет в БД молча.
22. SYSTEM_PROMPT routing rules (4)/(5): на «занеси/добавь/запиши»
    — звать `propose_*`, описание передавать VERBATIM, категорию
    не угадывать.

### G. Model upgrade
23. `gpt-4.1-nano` → `gpt-4.1-mini` как дефолт (.env, .env.example,
    docker-compose.yml). Mini корректно делает арифметику над
    tool-результатами (242 630 ₽ vs nano-выдумывал «24 263»),
    chains tool-вызовы, держит factual tone.
24. Pricing-словарь в `openai_provider.py` расширен: mini, full
    `gpt-4.1`. `/ai/usage` теперь корректно считает `est_cost_usd`
    на mini.

### H. Forecast math precomputed (consistency across follow-ups)
25. `get_forecast` возвращает `savings_capacity_monthly_cents` и
    `savings_capacity_daily_cents` напрямую — модель не выводит
    save-rate из spend-rate (это было причиной плывущих чисел в
    follow-up'ах: «1 320 ₽/день» / «39 000 за 25 дней» / «47 300 в
    месяц» — три инконсистентных ответа подряд).

### I. Dev tooling
26. Idempotent auto-seed (`app/dev_seed.py`) при `DEV_MODE=true`:
    owner + период + 9 категорий + 7 sample-транзакций. Переживает
    `./scripts/run-integration-tests.sh` cycle (merge-only, не
    клобит данные пользователя).

## Non-goals

- Multi-user / multi-tenant (по-прежнему single-tenant).
- Reasoning-модели (`o1`, `o3`) — overkill, mini достаточен.
- Provider-switching на Gemini / Anthropic — отложено.
- UI для редактирования synonyms категорий — пока хардкод-словарь
  для seed-имён.
- Schema migration для `category.synonyms` колонки — отложено
  (хардкод словарь покрывает 9 seed-категорий).

## Success criteria

- [x] api отказывается стартовать без `OPENAI_API_KEY` (RuntimeError
      на boot).
- [x] Live `POST /ai/chat` с tool-вызовом возвращает связный текст

      согласованные числа (всё считается из одного `get_forecast`

      ответах пользователю.

      командой `docker compose up`.
