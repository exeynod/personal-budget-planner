# Plan 04: Settings backend + Frontend integration

## Objective
Добавить user-level toggle enable_ai_categorization в AppUser (migration + service + API),
интегрировать AI-suggestion box в ActualEditor, добавить toggle в SettingsScreen.

## Tasks

### Task 1: Migration 0005 — AppUser.enable_ai_categorization
- Создать alembic/versions/0005_add_enable_ai_categorization.py
- ALTER TABLE app_user ADD COLUMN enable_ai_categorization BOOLEAN NOT NULL DEFAULT TRUE
- down: DROP COLUMN

### Task 2: AppUser модель
- Добавить enable_ai_categorization: Mapped[bool] = mapped_column(Boolean, default=True, nullable=False, server_default="true") в AppUser в app/db/models.py

### Task 3: Settings service
- Добавить get_enable_ai_categorization(db, tg_user_id) -> bool в app/services/settings.py
- Добавить update_enable_ai_categorization(db, tg_user_id, value) -> bool в app/services/settings.py

### Task 4: Settings schemas + route
- В app/api/schemas/settings.py:
  - SettingsRead: добавить enable_ai_categorization: bool
  - SettingsUpdate: добавить enable_ai_categorization: Optional[bool] = None
- В app/api/routes/settings.py:
  - GET: добавить enable_ai_flag = await settings_svc.get_enable_ai_categorization(...)
  - PATCH: добавить if body.enable_ai_categorization is not None: await update_enable_ai_categorization(...)
  - SettingsRead response: добавить enable_ai_categorization=...

### Task 5: Frontend types + api
- В frontend/src/api/types.ts:
  - SettingsRead: добавить enable_ai_categorization: boolean
  - SettingsUpdatePayload: добавить enable_ai_categorization?: boolean
  - Новый интерфейс: AiSuggestResponse { category_id: number | null; name: string | null; confidence: number }
- В frontend/src/api/ai.ts:
  - Добавить async function suggestCategory(q: string): Promise<AiSuggestResponse>
  - GET /api/v1/ai/suggest-category?q=<text>

### Task 6: useAiCategorize hook
- Создать frontend/src/hooks/useAiCategorize.ts
- Принимает: description: string, kind: CategoryKind, enabled: boolean
- Логика: debounce 500ms, min length 3 chars, вызывает suggestCategory(description)
- Возвращает: { suggestion: AiSuggestResponse | null, loading: boolean }
- Если enabled=false или description.length < 3 — возвращает suggestion=null

### Task 7: ActualEditor — AI suggestion box
- В frontend/src/components/ActualEditor.tsx:
  - Добавить пропс aiEnabled?: boolean (по умолчанию false — безопасный fallback)
  - useAiCategorize hook на description + kind
  - Если suggestion && suggestion.category_id !== null && aiEnabled:
    - Показывать AI suggestion box вместо select (фиолетовый #a78bfa стиль)
    - Содержимое: имя категории + confidence bar (linear-gradient фиолетовый)
    - Кнопка «Сменить» → скрывает suggestion, показывает обычный select
    - setCategoryId(suggestion.category_id) автоматически
  - Иначе: обычный select (текущее поведение)
- В frontend/src/components/ActualEditor.module.css:
  - .aiSuggestion: border-left 3px solid #a78bfa, background rgba(167, 139, 250, 0.08), padding, border-radius
  - .aiConfidenceBar: height 4px, background linear-gradient(to right, #a78bfa <confidence>%, #e5e7eb)
  - .aiSwitchBtn: button style для «Сменить»
