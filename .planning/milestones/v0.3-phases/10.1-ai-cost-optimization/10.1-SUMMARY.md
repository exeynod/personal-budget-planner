# Phase 10.1: AI Cost Optimization — Summary

**One-liner:** Reduced AI input cost ~50–60% via English prompts + tighter context window + categorization fixes; added per-call usage logging and `GET /ai/usage` cost observability endpoint.

## Why this phase

Pre-ПСИ AI cost review of v0.3 features (Phases 9, 10) found that Cyrillic system prompt + history=20 + redundant calls were burning input tokens unnecessarily. Single-tenant pet-app on a tight budget — every call must be optimal. INSERTED phase between Phase 10 (AI Categorization) and v0.3 closeout.

## Wave-by-wave

### Wave 1 — Token reduction (commit `e7589dc`)

| Task | What | Effect |
|------|------|--------|
| T-1.1 | `SYSTEM_PROMPT` translated to English + explicit "Reply in Russian" | ~80 → ~50 tokens (~38% drop) |
| T-1.2 | `TOOLS_SCHEMA` descriptions and parameter docs to English | ~200 → ~80 tokens (~60% drop) |
| T-1.3 | `AI_MAX_CONTEXT_MESSAGES` 20 → 8 | ~60% smaller history payload |
| T-1.4 | Drop decorative `cache_control` block | Cleaner code; no behavioural change (prompt < 1024 tok) |

### Wave 2 — Observability + T-3.1 (commit `7205c4a`)

| Task | What |
|------|------|
| T-2.1 | Capture OpenAI stream `usage` chunk via `stream_options.include_usage`; emit `{type: "usage", data: {...}}` provider event |

| T-2.3 | New `GET /api/v1/ai/usage` endpoint — today / session_total aggregates from in-process ring buffer |
| T-3.1 | Rate limit `_RATE_LIMIT` 30 → 10 req/min (cost-control ceiling) |

### Wave 4 — Categorization fixes (commit `2224d15`)

| Task | What |

## Deviations from plan

### T-3.3 SKIPPED — tool-result caching

**Found during:** plan re-evaluation (Wave 3 design).
**Issue:** Caching tool-result on `tool_call_complete` would only skip the local DB query (~ms), not the second LLM round-trip — the LLM still needs to verbalize the cached result. The expensive part is the LLM call, not the tool function.
**Decision:** SKIP. Documented in plan + this summary. False win.

## Known stubs / follow-ups

- Usage records are in-process: counters reset on `api` container restart. If multi-process visibility is ever needed (e.g. multi-tenant migration), materialize `_usage_buffer` to a `ai_usage` table.
- Provider-switching to Gemini Flash 2.0 (~25% cheaper than nano) deferred to Phase 11+ if cost still pressures.
- LRU cache has no TTL — embeddings are deterministic per input, so this is correct, but if `text-embedding-3-small` is ever swapped for a different model the per-process cache should be cleared on startup (already happens — fresh OrderedDict).
