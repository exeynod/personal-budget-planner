# Phase 10.1: AI Cost Optimization (INSERTED)

**Inserted after:** Phase 10
**Reason:** Pre-ПСИ cost review of v0.3 AI features (Phases 9, 10) revealed
~57% of AI input tokens are wasted on Cyrillic system prompt, oversized
history window, and redundant calls. User on tight budget — every call
must be optimal.

**Source:** ad-hoc review session 2026-05-06; full analysis preserved
in commits + this phase's SUMMARY/VERIFICATION.

## Goal

Reduce AI input cost by **~50–60%** without changing user-facing behaviour.
Add observability so future cost regressions are detectable.

## Scope

11 discrete fixes across the AI surface — chat, categorization, and infra:

1. Translate `SYSTEM_PROMPT` and all `TOOLS_SCHEMA` descriptions to English
   (LLM still answers in Russian per explicit instruction). Cyrillic
   tokenizes ~2.3× more tokens than Latin in `gpt-4.1-nano`.
2. `AI_MAX_CONTEXT_MESSAGES` 20 → 8. Sliding window covers 3–4 user/AI
   pairs — sufficient context for budget questions.
3. Add per-conversation tool-result cache (60s TTL) to avoid repeated
   `get_period_balance` / `get_category_summary` calls within one session.
4. Remove decorative `cache_control` block from system prompt (effectively
   a no-op — OpenAI caches only ≥1024 tokens, current prompt is ~425).
5. Log `usage` (prompt/completion/cached tokens + estimated USD) from
   OpenAI streaming chunks for cost observability.
6. `Query(min_length=3)` on `/ai/suggest-category` — frontend already
   filters but backend was at 1.
7. Rate limit 30 → 10 req/min for chat (cost-control ceiling).
8. Generate embedding when **creating** a category (currently only on
   PATCH — bug: new categories invisible to suggest-category until rename).
9. Skip embedding refresh when PATCH name == old name.
10. Async LRU cache on `embed_text` (latency + dedupe wins; cost win
    negligible at $0.02/M).
11. New `GET /api/v1/ai/usage` endpoint — daily/monthly token totals
    and estimated cost from logged `usage` records.

## Non-goals

- **Provider switching.** Gemini Flash 2.0 ~25% cheaper but requires new
  `GeminiProvider` impl + testing — defer to Phase 11+ if cost still
  pressures.
- **Anthropic / DeepSeek.** Both more expensive than nano for this
  workload.
- **Prompt-caching trick** (padding system to ≥1024 tokens). The 75%
  discount on cached input only beats English-translation savings at
  high frequency (>1k req/day) which we're nowhere near.

## Success criteria

- [ ] Token usage drops measurably on a fixed test prompt: same Q&A
      pre/post — input tokens reduced by 50%+.
- [ ] Cost logger writes `ai.usage` structured log on every chat call.
- [ ] `/ai/usage` endpoint returns daily/monthly aggregates.
- [ ] All existing tests pass (`./scripts/run-integration-tests.sh tests/` → 0 failed).
- [ ] No user-facing behavioural change (LLM still replies in Russian,
      same suggestion box, same chat UI).

## Dependencies

- Phase 9 (AI Assistant) — chat infra exists.
- Phase 10 (AI Categorization) — embedding infra exists.

## Out-of-scope artifacts

- No new ORM tables (usage logging goes through structured logs first;
  if `/ai/usage` proves valuable, materialize to DB in Phase 11).
- No DB migration needed.
