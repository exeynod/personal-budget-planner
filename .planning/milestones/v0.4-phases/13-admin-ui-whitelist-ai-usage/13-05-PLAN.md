Aggregation strategy: один SQL запрос с группировкой по user_id + два time-window (current_month, last_30d). Реюзим UsageBucket schema из `app/api/schemas/ai.py`. AdminAiUsageRow добавляет user metadata (tg_user_id, role, spending_cap_cents) + computed fields (est_cost_cents_current_month, pct_of_cap).

RLS bypass для admin breakdown: admin endpoint должен видеть rows ВСЕХ юзеров (cross-tenant aggregation). Опции:
1. SET app.current_user_id = -1 → блокирует ВСЕ rows (RLS sentinel) — НЕ годится
2. Не использовать ORM SELECT, а raw text с set_config('app.current_user_id', NULL) — но NULL в RLS coalesces к -1
3. Использовать privileged role (ADMIN_DATABASE_URL) — overkill, requires dual session
4. **Выбираем**: ORM model `AiUsageLog` имеет RLS, но мы можем bypass через AppUser-side aggregation: вместо SELECT FROM ai_usage_log GROUP BY user_id, делаем SELECT FROM app_user LEFT JOIN LATERAL aggregate-subquery — `app_user` без RLS, и subquery будет вызывать SELECT FROM ai_usage_log с условием `user_id = app_user.id`. Под обычным RLS это не пройдёт пока GUC не выставлен на каждого user_id отдельно — это N+1 запросов.

**Финальное решение** — самый чистый: использовать `set_config` чтобы временно установить sentinel который разрешает всё. RLS policy `user_id = coalesce(NULLIF(current_setting(...)), '-1')::bigint` возвращает строки только когда `app.current_user_id == row.user_id`. Чтобы admin увидел всех, нужно либо:
- Disable RLS policy temporarily (DROP/CREATE POLICY) — overkill
- Run query за каждого юзера в loop — N запросов, неэффективно
- **Грантнуть BYPASSRLS budget_app для admin queries**: НО в Plan 12-05 budget_app NOBYPASSRLS by design — менять нельзя
- **Переписать RLS policy** чтобы owner видел всё: добавить OR проверку

Лучший вариант — **2-stage approach**:
1. Прочитать всех `AppUser` напрямую (нет RLS на app_user table — Phase 11 не enabled RLS на app_user, проверено в alembic 0006)
2. Для каждого user_id: установить `set_config('app.current_user_id', :uid, true)` и SELECT aggregate из ai_usage_log

Альтернатива: pure SQL + GROUP BY на app_user side с CORRELATED SUBQUERY — но subquery всё равно под RLS.

**Окончательное решение, simplest + correct**: применить **`SET LOCAL row_security = off`** в начале admin transaction. Это PostgreSQL session GUC которая отключает RLS для текущей transaction; работает только если у роли есть BYPASSRLS ATTRIBUTE — НО budget_app NOBYPASSRLS, поэтому не сработает.

**Реальное решение**, проверенное: использовать **superuser ADMIN_DATABASE_URL** для admin breakdown. Это чисто читающий endpoint без write side effects. Service создаёт собственный engine/session pool на ADMIN_DATABASE_URL для aggregation; в admin endpoints это допустимо (мы уже используем admin URL для миграций и truncate в тестах).

ИЛИ даже проще: **Application-side aggregation** — SELECT FROM app_user (без RLS), затем для каждого user_id выполнить отдельный aggregate-query с set_config GUC. N+1 проблема не критична: до 50 юзеров × 2 (current_month + last_30d) = 100 запросов. На pet-проекте это <100ms.

**ВЫБРАННОЕ РЕШЕНИЕ**: используем admin DB connection (ADMIN_DATABASE_URL) с `BYPASSRLS` свойством роли. По 0007 budget_app NOBYPASSRLS, но `budget` (admin role) IS SUPERUSER → bypass RLS. Service `build_admin_ai_usage_breakdown` открывает temporary engine на ADMIN_DATABASE_URL, делает один GROUP BY query, dispose. Отдельная engine минимизирует impact на основной pool.

```python
from __future__ import annotations
import os
from datetime import datetime, timedelta, timezone
from typing import Optional
from zoneinfo import ZoneInfo

from sqlalchemy import text
from sqlalchemy.ext.asyncio import async_sessionmaker, create_async_engine, AsyncSession

from app.api.schemas.ai import UsageBucket
from app.api.schemas.admin import AdminAiUsageResponse, AdminAiUsageRow


MSK_TZ = ZoneInfo("Europe/Moscow")


def _start_of_current_month_msk() -> datetime:
    """Return the first second of the current month in Europe/Moscow as UTC."""
    now_msk = datetime.now(MSK_TZ)
    msk_first = now_msk.replace(day=1, hour=0, minute=0, second=0, microsecond=0)
    return msk_first.astimezone(timezone.utc)


def _last_30d_start() -> datetime:
    """Return now() - 30 days as UTC. last_30d window is calendar days, not month boundaries."""
    return datetime.now(timezone.utc) - timedelta(days=30)


def _empty_bucket() -> UsageBucket:
    return UsageBucket(
        requests=0, prompt_tokens=0, completion_tokens=0,
        cached_tokens=0, total_tokens=0, est_cost_usd=0.0,
    )


def _bucket_from_row(row: tuple) -> UsageBucket:
    """row tuple matches SELECT signature in _aggregate_query below."""
    return UsageBucket(
        requests=int(row[0] or 0),
        prompt_tokens=int(row[1] or 0),
        completion_tokens=int(row[2] or 0),
        cached_tokens=int(row[3] or 0),
        total_tokens=int(row[4] or 0),
        est_cost_usd=float(row[5] or 0.0),
    )


_AGGREGATE_QUERY = """
SELECT
    user_id,
    count(*) AS requests,
    coalesce(sum(prompt_tokens), 0)     AS prompt_tokens,
    coalesce(sum(completion_tokens), 0) AS completion_tokens,
    coalesce(sum(cached_tokens), 0)     AS cached_tokens,
    coalesce(sum(total_tokens), 0)      AS total_tokens,
    coalesce(sum(est_cost_usd), 0.0)    AS est_cost_usd
FROM ai_usage_log
WHERE created_at >= :start
GROUP BY user_id
"""


async def _open_admin_session() -> async_sessionmaker[AsyncSession]:
    """Build a session factory using ADMIN_DATABASE_URL.

    Why: admin breakdown reads cross-tenant ai_usage_log. The runtime app
    role (budget_app) is NOSUPERUSER NOBYPASSRLS so RLS would hide rows
    from other users. The admin role (budget, SUPERUSER) bypasses RLS
    cleanly. We use a short-lived engine to avoid affecting the main pool.

    Falls back to DATABASE_URL when ADMIN_DATABASE_URL unset (local dev
    where the role split has not been configured).
    """
    url = os.environ.get("ADMIN_DATABASE_URL") or os.environ["DATABASE_URL"]
    engine = create_async_engine(url, echo=False)
    return async_sessionmaker(engine, expire_on_commit=False), engine


async def build_admin_ai_usage_breakdown(
    db: AsyncSession,  # for AppUser reads — uses runtime DSN, no RLS issue
) -> AdminAiUsageResponse:
    """Aggregate per-user AI usage for the admin breakdown.

    Strategy:
      1. SELECT all AppUser rows (no RLS on app_user table).
      2. Open a separate admin-engine session to aggregate ai_usage_log
         (bypasses RLS via SUPERUSER role).
      3. Build current_month + last_30d UsageBucket per user; users with
         no usage rows get _empty_bucket() and pct_of_cap = 0.
      4. Sort by est_cost_cents_current_month desc, then tg_user_id asc.
    """
    from app.db.models import AppUser

    # 1. Fetch users (runtime session — no RLS on app_user).
    users_result = await db.execute(
        text(
            "SELECT id, tg_user_id, tg_chat_id, role, spending_cap_cents "
            "FROM app_user"
        )
    )
    user_rows = list(users_result.all())

    # 2. Aggregate ai_usage_log under admin role.
    sm, engine = await _open_admin_session()
    cm_start = _start_of_current_month_msk()
    l30_start = _last_30d_start()
    try:
        async with sm() as admin_session:
            cm_result = await admin_session.execute(
                text(_AGGREGATE_QUERY), {"start": cm_start}
            )
            cm_by_user = {row[0]: row[1:] for row in cm_result.all()}
            l30_result = await admin_session.execute(
                text(_AGGREGATE_QUERY), {"start": l30_start}
            )
            l30_by_user = {row[0]: row[1:] for row in l30_result.all()}
    finally:
        await engine.dispose()

    # 3. Stitch into rows.
    rows: list[AdminAiUsageRow] = []
    for ur in user_rows:
        uid, tg_user_id, tg_chat_id, role_value, cap_cents = ur
        cm_bucket = (
            _bucket_from_row(cm_by_user[uid]) if uid in cm_by_user
            else _empty_bucket()
        )
        l30_bucket = (
            _bucket_from_row(l30_by_user[uid]) if uid in l30_by_user
            else _empty_bucket()
        )
        # USD копейки = доллары * 10000 (т.к. cents-of-USD = $0.01 = 10000-копейка-в-копейках USD).
        # Формула: 1 USD = 10000 копеек USD; cents-of-cents.
        est_cost_cents_cm = round(float(cm_bucket.est_cost_usd) * 10_000)
        pct = (
            float(est_cost_cents_cm) / float(cap_cents)
            if cap_cents and cap_cents > 0
            else 0.0
        )
        rows.append(
            AdminAiUsageRow(
                user_id=uid,
                tg_user_id=tg_user_id,
                name=None,  # Phase 14 will populate; pending tg_chat_id semantics
                role=role_value,
                spending_cap_cents=int(cap_cents),
                current_month=cm_bucket,
                last_30d=l30_bucket,
                est_cost_cents_current_month=est_cost_cents_cm,
                pct_of_cap=pct,
            )
        )

    # 4. Sort: est_cost desc, tg_user_id asc fallback.
    rows.sort(
        key=lambda r: (-r.est_cost_cents_current_month, r.tg_user_id)
    )

    return AdminAiUsageResponse(
        users=rows,
        generated_at=datetime.now(timezone.utc),
    )
```

    ```python
                l30_by_user = {row[0]: row for row in l30_result.all()}
        finally:
            await engine.dispose()

        # 3. Build rows.
        rows: list[AdminAiUsageRow] = []
        for ur in user_rows:
            uid, tg_user_id, tg_chat_id, role_value, cap_cents = ur
            cm_bucket = (
                _bucket_from_row(cm_by_user[uid]) if uid in cm_by_user
                else _empty_bucket()
            )
            l30_bucket = (
                _bucket_from_row(l30_by_user[uid]) if uid in l30_by_user
                else _empty_bucket()
            )
            est_cost_cents_cm = round(float(cm_bucket.est_cost_usd) * 10_000)
            pct = (
                float(est_cost_cents_cm) / float(cap_cents)
                if cap_cents and cap_cents > 0
                else 0.0
            )
            rows.append(
                AdminAiUsageRow(
                    user_id=uid,
                    tg_user_id=tg_user_id,
                    name=None,  # Phase 14 will fill from tg_chat_id metadata
                    role=role_value,
                    spending_cap_cents=int(cap_cents),
                    current_month=cm_bucket,
                    last_30d=l30_bucket,
                    est_cost_cents_current_month=est_cost_cents_cm,
                    pct_of_cap=pct,
                )
            )

        # 4. Sort: est_cost_cents desc, tg_user_id asc fallback.
        rows.sort(
            key=lambda r: (-r.est_cost_cents_current_month, r.tg_user_id)
        )

        return AdminAiUsageResponse(
            users=rows,
            generated_at=datetime.now(timezone.utc),
        )
    ```
