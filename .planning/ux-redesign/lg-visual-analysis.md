# Liquid Glass — визуальный разбор референсов владельца

Референсы (owner, «вайб»): `lg-refs/ref1-payrix-overview.png`, `ref2-spending-budget.png`,
`ref3-nutrition-dashboard.png`, `ref4-ios26-tabbar.png`. Анализ — мой (vision), т.к.
WebFetch/агенты не видят изображения. Текущая реализация (плоская аврора + простые
frosted-карточки) владельцем отклонена как «херня» — слишком плоско/тускло.

## Что делает «настоящий» Liquid Glass на этих рефах

### 1. Фон — насыщенный, объёмный, «жидкое стекло» (НЕ плоский градиент)

- **ref1 (Payrix):** фон = фотографичный 3D-glass рендер: толстое цветное стекло
  (бирюза→оранж) с преломлением, бликами, каустиками, мягкими тенями. Глубина и
  «оптика», а не CSS-радиалки. Заголовок крупный белый поверх стекла.
- **ref2 (Spending Budget):** мягкая пастельная аврора (голубой→персик→крем), очень
  диффузная, светлая. Тоже с ощущением света, но спокойнее ref1.
- Вывод: нужен **богатый фон** — либо реальная glass-картинка/3D-blob-рендер (как ref1),
  либо гораздо более насыщенная, «оптическая» мешевая аврора с бликами (не мои плоские
  4 радиалки). Возможны 2 варианта фона на выбор владельца.

### 2. Карточки — frosted glass с глянцем и световым кантом

- Высокая прозрачность + сильный backdrop-blur + **saturation boost** (цвет фона
  просвечивает насыщенно).
- **Тонкая яркая верхняя кайма** (specular top-edge highlight) — `border` или
  `box-shadow inset` со светлым rgba сверху, тёмнее снизу → стекло «ловит свет».
- Мягкая **внешняя тень + лёгкий внутренний glow**. Скруг ~20–28px (крупнее моего 14).
- ref2: карточки почти белые-молочные, очень чистые, с тончайшим бордером.

### 3. Таб-бар — плавающая стеклянная «пилюля» с линзингом (iOS 26)

- **ref4 (Apple Music):** таб-бар = floating capsule (не edge-to-edge), отстоит от краёв,
  с **реальным преломлением** — контент за стеклом искажается/увеличивается под
  «линзой» активного таба, яркий specular-кант по периметру.
- Моя версия (edge-to-edge плоская панель) → заменить на **floating glass pill** с
  ярким кантом, тенью, blur; активный таб — с подсветкой/«линзой».

### 4. Кнопки действий — круглые glass-кнопки

- ref1: Pay / Add money / Transfer = круглые frosted-кнопки с иконкой. Применимо к нашему
  «+», фильтру, и быстрым действиям.

### 5. Дата-виз (для план/факт/лимиты) — из ref2/ref3

- **ref2:** вертикальный **гейдж/линейка** с маркером (красная риска = факт vs шкала
  лимита) — идеально под «лимит категории и сколько потрачено».
- **ref3:** **кольцевые donut-прогрессы** (%), **сегментные мультицветные бары**
  (потрачено / осталось / таргет), **streak-сетка**. Палитра blue/teal/purple.
- Вывод: для категорий/плана — кольца или сегментные бары + гейдж лимита вместо
  плоских строк «385,18 / 16 000».

## Конкретное направление апгрейда (web native shell)

1. **Фон:** заменить плоские радиалки на насыщенный «оптический» мешевый фон (или
   glass-картинку) с бликами; притушить под контентом для читаемости.
2. **Карточки:** добавить specular top-border + inner-glow + крупнее радиус; усилить
   blur+saturate; многослойные тени.
3. **Таб-бар:** floating glass pill с кантом и активной «линзой» (refraction —
   приблизить через layered translucency + bright rim; полноценный лензинг в CSS
   ограничен, но визуально приблизим).
4. **Кнопки:** круглые glass-кнопки с кантом.
5. **Дата-виз:** кольца/сегментные бары/гейдж лимита для план-факт (увязать с UX-реворком
   планирования — см. RESEARCH.md §3).
6. Сохранить читаемость финданных (тёмный ink, достаточный контраст) и НЕ регрессить
   Maximal Poster.

## Что ещё полезно прислать

- Если есть — реальные **PNG/исходники** фоновых glass-рендеров (как ref1), чтобы
  использовать их как ассет фона (CSS не воспроизведёт фотографичное стекло 1:1).
- Предпочтение по фону: «оптический» цветной (ref1) vs мягкий пастельный (ref2).
