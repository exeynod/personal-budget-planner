### HIGH

**H-1: Phase 14 require_onboarded gate каскадная регрессия v0.2/v0.3 тестов** ✓ FIXED
- **Severity:** HIGH (was)
- **Status:** **FIXED inline** (test-infra only, никаких изменений business code)
- **Component:** `tests/helpers/seed.py:seed_user`, `tests/conftest.py:single_user/two_tenants`, 14 fixture files
- **Symptom:** 121 теста из v0.2/v0.3 фейлили `assert 409 == 200/422/404` после Phase 14 wired `require_onboarded` на 10 routers
- **Root cause:** Phase 14 не обновил existing fixture defaults — `seed_user(onboarded_at=None)` оставил всех seeded users в pre-onboarded state
- **Fix applied:** Sentinel-default `onboarded_at=datetime.now(UTC)` в `seed_user`; explicit `onboarded_at=None` в `seed_two_role_tenants` для invite-flow семантики; 11 fixture files напрямую пропатчены regex'ом; 4 bootstrap-via-/me files получили `UPDATE app_user SET onboarded_at = NOW()` после `/me` call
- **Result:** 326 passed → улучшение с 222 → +104 тестов проходят
- **Lesson:** Phase 14 SUMMARY должна была включать «test infrastructure migration» step, не только code change

**H-2: tests/test_embedding_backfill.py — mock spec breaks DB writes**
- **Severity:** HIGH (test bug, not business code)
- **Component:** `tests/test_embedding_backfill.py:70-76` (`_make_embed_service_mock`)
- **Symptom:** 4 теста асёртят `row_count == N` для `category_embedding` table, но row count = 0; `count` (return value) корректен
- **Root cause:** `MagicMock(spec=EmbeddingService)` подменяет ВСЕ методы класса, включая `upsert_category_embedding` → mock не пишет в DB; helper возвращает `written += 1` для каждого «успешного» mock-call, но БД пуста
- **Proposed fix:**
  ```python
  # Mock только embed_texts, оставить остальные методы реальными
  with patch.object(EmbeddingService, "embed_texts", new=AsyncMock(side_effect=fake_embed)):
      count = await backfill_user_embeddings(db, user_id=user.id)
  ```
- **Owner:** v0.5 mini-phase (тест-only fix)

**H-3: ai_usage_log.est_cost_usd Float (pre-existing technical debt)**
- **Severity:** HIGH (deferred to mini-phase per audit)
- **Component:** `app/db/models.py` AiUsageLog
- **Symptom:** Column type Float нарушает CLAUDE.md «Money: BIGINT kopecks (*_cents fields), no Float»
- **Status:** Документировано в `v0.4-MILESTONE-AUDIT.md:tech_debt`. Ожидает отдельной mini-phase для миграции `est_cost_usd Float → cost_cents BIGINT` с alembic backfill
- **Workaround in code:** `admin_ai_usage.py` конвертирует через `round(est_cost_usd * 100)` (CR-01 fix); spend_cap.py использует `ceil(SUM(est_cost_usd) * 100)`
- **Owner:** v0.5

### MEDIUM

**M-1: WR-01 Double DB session в enforce_spending_cap**
- **Severity:** MEDIUM (deferred architectural decision)
- **Component:** `app/api/dependencies.py:enforce_spending_cap` + `app/api/routes/ai.py`, `ai_suggest.py`
- **Symptom:** Каждый `/ai/chat` запрос открывает 2 сессии: `Depends(get_db)` для cap-проверки + `Depends(get_db_with_tenant_scope)` для handler body
- **Impact:** Pool pressure под нагрузкой (2×). PostgreSQL default 25 connections → ~12 concurrent /ai/chat достигнут лимит
