# UX Pipeline — TG Budget Planner (веб, тема Liquid Glass / native shell)

Полный UX-пайплайн текущего веб-дизайна для передачи дизайнеру. Все скриншоты
сняты с **живого** инстанса (Vite `localhost:5173` → backend `:8000`, DEV-режим
авто-логина владельца, наполненная БД: 10 категорий, 19 факт-трат, 11 plan-строк,
4 подписки, доход 180 000 ₽, активный период). Тема — `liquid_glass` (нативная
iOS-эстетика: крупные заголовки, inset-grouped карточки, bottom-sheets,
action-sheets, нижний таб-бар).

Viewport скриншотов: **390 × 844** (mobile / Telegram Mini App), 2× DPR.
Скриншоты лежат в [`screens/`](screens/), именование `NN-screen-state.png`.

---

## 0. Глоссарий состояний (нотация)

| Нотация                     | Значение                                                             |
| --------------------------- | -------------------------------------------------------------------- |
| **tab-root**                | экран — корень таба (большой заголовок, без back-кнопки)             |
| **push**                    | экран, втолкнутый поверх (есть `‹ Назад`, native push-анимация)      |
| **sheet**                   | модальный bottom-sheet (drag-handle, затемнённый фон, drag-to-close) |
| **action-sheet**            | iOS-стиль список действий снизу + «Отмена»                           |
| **picker**                  | sheet выбора одного значения (категория, дата)                       |
| **inline-editor**           | редактор внутри уже открытого sheet (без нового слоя)                |
| **segment**                 | сегмент-контрол (Расходы/Доходы, ДЕНЬ/НЕД./КАТ.)                     |
| **empty / loading / error** | пустое / загрузка / ошибка                                           |

**Знак дельты — «+ = хорошо».** Расходы: `Лимит − Факт`. Доходы: `Факт − План`.
Отрицательная дельта («В запасе») красная, положительная — зелёная/нейтральная.

## 1. Карта навигации

```
┌─ AUTH / GATE (поверх всего, до выбора темы) ──────────────────────────┐
│   loading «ЗАГРУЗКА…» [43]      error «ОШИБКА / ПОВТОРИТЬ» [42]         │
└───────────────────────────────────────────────────────────────────────┘
        │ /me.onboarded_at == null → ОНБОРДИНГ (одноэкранный) [34–40]
        │ /me.onboarded_at != null → ГЛАВНЫЙ SHELL ↓
        ▼
NATIVE SHELL (нижний таб-бар, 4 таба + центральный путь через «+» на Главной)
 ├─ [Таб] Главная (home)            ← tab-root, корень роутера
 │    ├─ «+» (хедер) ─────────────► sheet AddSheet «Новая транзакция» (факт)
 │    ├─ карточка «План месяца» ──► push План месяца
 │    └─ строка категории ────────► push Категория-деталь
 ├─ [Таб] Транзакции (txn)          ← push поверх роутера
 │    └─ кнопка фильтра ──────────► action-sheet «Фильтр транзакций»
 ├─ [Таб] AI (ai)                   ← push, чат-композер
 └─ [Таб] Управление (mgmt)         ← push, хаб-список
      ├─ Шаблон бюджета ───────────► push Шаблон (+ disclosure «Строки»)
      ├─ Аналитика ────────────────► push Аналитика (бар-чарт)
      ├─ Подписки ─────────────────► push Подписки ──► sheet «Меню подписки»
      ├─ Настройки ────────────────► push Настройки (+ «Привести остаток»)
      └─ Доступ (owner-only) ──────► push Доступ
```

**Таб-бар:** `Главная` (House) · `Транзакции` (ListBullets) · `AI` (Sparkle) ·
`Управление` (GearSix). Активный таб подсвечен акцентом. Тап по `Главная`
делает `popToRoot()` (сбрасывает стек). Остальные табы — `push` поверх. FAB как
такового нет: ввод факт-траты открывается «+» в хедере Главной (и кнопкой
«Добавить в план» — на экране Плана).

**Жесты:** sheet/action-sheet закрываются свайпом вниз (drag-handle), тапом по
затемнённому фону, либо кнопкой «Отмена». Push-экраны — `‹ Назад`.

## 2. Экраны

### 2.1 Главная (Home) — tab-root

![Главная — Расходы](screens/01-home-expenses.png)
![Главная — Доходы](screens/02-home-income.png)

**Назначение.** Стартовый экран: остаток на счёте + актуальная дельта план/факт,
список категорий с дельтами.

**Ключевые элементы.**

- Большой заголовок «Главная», «+» справа (→ AddSheet).
- Карточка **«Остаток на счёте»** + 4-уровневый ладдер:
  `ЛИМИТ · РАСПИСАНО · ФАКТ · В ЗАПАСЕ` (Лимит = Σ лимитов категорий за период;
  Расписано = Σ непроведённых plan-строк; Факт = Σ факт-расходов;
  В запасе = Лимит − Факт, красная если отрицательна).
- Карточка-кнопка **«План месяца»** (→ push План).
- **Segment «Расходы / Доходы»** — переключает список категорий ниже.
- Список категорий (иконка + имя + «Без плана»/дельта + сумма). У каждой —
  тап → Категория-деталь.

**Состояния.** Расходы (по умолчанию) [01] / Доходы [02]. Категории «Без плана»
показывают подпись-плейсхолдер.

**Взаимодействия.**

- «+» → sheet AddSheet (факт).
- Карточка «План месяца» → push План.
- Segment → смена набора категорий (без навигации).
- Строка категории → push Категория-деталь.
- Таб «Главная» повторно → popToRoot.

### 2.2 AddSheet «Новая транзакция» — sheet (ввод факт-траты)

![Пустой](screens/03-addsheet-fact-empty.png)
![С суммой](screens/04-addsheet-fact-amount.png)
![Пикер категории](screens/05-addsheet-category-picker.png)
![Дата — action-sheet](screens/06-addsheet-date-actionsheet.png)
![Дата — календарь](screens/07-addsheet-date-calendar.png)
![Готова к отправке](screens/08-addsheet-fact-ready.png)
![Подтверждение отмены](screens/09-addsheet-cancel-confirm.png)

**Назначение.** Быстрый ввод факт-траты в один проход (core value продукта).

**Ключевые элементы.**

- Хедер: «Отмена» (слева) · заголовок «Новая транзакция».
- Крупный дисплей суммы `0 ₽` (растёт при наборе на keypad).
- Поле «Описание (кафе / продукты / …)».
- Строка **«Категория · Выбрать ›»** (→ picker).
- Строка **«Дата · Сегодня ›»** (→ action-sheet).
- Цифровой **keypad** (1–9, `,`, 0, backspace).
- Primary CTA снизу: `Введите сумму` (disabled) → `Добавить` (enabled, когда
  сумма > 0 и выбрана категория).

**Состояния / поток.**

1. Пустой [03] — CTA «Введите сумму» неактивна.
2. После набора суммы [04] — дисплей показывает «1 250 ₽».
3. **Picker категории** [05] — sheet со списком категорий (savings/paused скрыты),
   single-select; выбор закрывает picker и подставляет имя в строку.
4. **Дата → action-sheet** [06] — «Сегодня / Вчера / Своя дата» + «Отмена».
5. **Своя дата → календарь** [07] — раскрывается inline-месячная сетка
   (`NativeCalendar`) прямо в action-sheet.
6. Готова [08] — сумма + категория заданы, CTA «Добавить» активна.
7. **Отмена при «грязной» форме** [09] — confirm-диалог (защита от потери ввода);
   при чистой форме закрывается сразу.

**Взаимодействия.**

- keypad → меняет сумму. «Категория»/«Дата» строки → соответствующий слой.
- Submit → создаёт факт (`createActualV10`), закрывает sheet, бампит refetch-token
  (Главная/Транзакции мгновенно перечитывают данные), авто-переключение периода
  если дата вне текущего.
- Закрытие: «Отмена» / свайп / фон / Esc.

### 2.3 План месяца (Plan) — push

![Расходы — сводка](screens/10-plan-expenses-summary.png)
![Детализация раскрыта](screens/11-plan-detail-expanded.png)
![Доходы](screens/12-plan-income.png)
![AddSheet «В план»](screens/13-plan-addsheet.png)
![AddSheet «В план» — календарь](screens/14-plan-addsheet-calendar.png)

**Назначение.** Распределение бюджета месяца по категориям; перевод
запланированных строк в факт («Провести»).

**Ключевые элементы.**

- Хедер `‹ Назад` · «План месяца».
- **Segment «Расходы / Доходы»**.
- Карточка **«Осталось распределить»** (профицит): зелёный «OK» бейдж при ≥ 0,
  красный — при перерасходе.
- CTA **«+ Добавить в план»** (→ AddSheet в режиме «В план»).
- Секция **«Регулярные»** — строки подписок/повторов с кнопкой «Провести»/«Отмена».
- Список категорий: имя + поле лимита (₽, редактируемое) + disclosure
  **«Детализация · N»** (раскрывает строки плана данной категории).

**Состояния.**

- Расходы (сводка) [10] — «Осталось распределить +179 000 ₽», категории с
  лимитами и счётчиком детализации.
- **Детализация раскрыта** [11] — список plan-строк категории; per-row ладдер
  Лимит/Расписано/Свободно; у каждой строки кнопка **«Провести»** (→ факт) или
  **«Отмена»** (если уже проведена); warn-строка при перерасходе.
- Доходы [12] — segment переключён, доходные категории с план-суммой и дельтой.

**Взаимодействия.**

- Поле лимита → правка → commit (upsert лимита категории).
- «Детализация» → аккордеон (single-open).
- «Провести» на строке → создаёт факт из плановой строки; «Отмена» → откат.
- «+ Добавить в план» → AddSheet «В план» [13] (тот же keypad/категория/дата UI,
  но заголовок «В план», CTA «Добавить в план», submit = `createPlanned`).
  Календарь даты работает идентично факт-режиму [14].
- `‹ Назад` → Главная.

## 3. Пользовательские сценарии (end-to-end)

1. **Онбординг → первый запуск.** Gate (loading [43]) → Онбординг [34–40]
   (доход, баланс, день цикла, пресеты, бот) → «Начать» → Главная [01].
2. **Ввод факт-траты (core value).** Главная [01] → «+» → AddSheet [03] →
   набор суммы [04] → выбор категории [05] → (опц.) дата [06]/[07] → «Добавить»
   [08] → sheet закрыт, Главная/ладдер обновлены мгновенно.
3. **Планирование месяца.** Главная → «План месяца» [10] → правка лимитов,
   «Детализация» [11] → «+ Добавить в план» [13]/[14]; Доходы-сегмент [12].
4. **Проведение плана в факт.** План [10] → «Детализация» [11] → «Провести» на
   строке → plan-строка становится фактом (дельта пересчитана).
5. **Подписки.** Управление [19] → Подписки [25] → «···» [26] → Сменить день
   [27] / Изменить цену [28] / Отменить [29].
6. **Reconcile (сверка остатка).** Управление [19] → Настройки [30] → «Привести
   остаток» [31] → ввод реального остатка [32] → «Привести» → корректирующая
   операция.
7. **Анализ.** Управление [19] → Аналитика [22] → переключение ДЕНЬ/НЕД./КАТ.
   [23]/[24] и месяцев.
8. **Деталь категории.** Главная [01] → строка категории → Категория-деталь [15]
   → «Поднять лимит» → План.

## 5. Что НЕ снято и почему

- **AI active/typing** — требует реального ответа LLM на живой запрос; снят только
  initial-экран.
- **empty-состояния** Категория-деталь / Шаблон / Подписки / loading-error
  Аналитики / Настроек / Доступа — БД наполнена, эти ветки недостижимы без
  спец-моков конкретных эндпоинтов (а часть из них защищена `require_onboarded`).
- **Тосты успеха** (после submit факта/плана) — короткоживущие, не попали в кадр;
  поведение описано в §2.2 (refetch + закрытие sheet).
- **Gate-состояния [42][43]** сняты через перехват `/me` (500 / задержка) — это
  единственные «синтетические» кадры; «Транзакции — пусто» [41] — через мок
  пустого списка операций.

Все остальные кадры — реальные интеракции на живом инстансе (тапы по строкам,
открытие sheets, переключение сегментов/периодов, набор на keypad).
