# Phase 10.2: AI Hardening + Write-Flow — Summary

**One-liner:** AI-surface запущен в живую среду впервые: pre-existing
баги Phase 9/10 закрыты, добавлен propose-and-approve write-flow,
переехали на gpt-4.1-mini, embedding-recall на коротких терминах
вырос с ~30% до 90%+ через synonyms.

## Why this phase

До этой сессии AI-surface (chat tool-calls, embedding refresh,
suggest-category, embedding-on-create) **никогда не выполнялся в
живой среде** — `OPENAI_API_KEY` не пробрасывался в api контейнер
со времён Phase 9. Тесты с моками не валидировали реальный OpenAI
contract / pgvector cast / AsyncSession-commit. Когда ключ
заработал — сразу всплыли 6 latent багов. Параллельно пользователь
запросил write-flow (AI добавляет транзакции через чат) и upgrade
модели для аналитики.

### F. Write-flow (`81dc25b`, `d3ead29` (drop hint))

Архитектура «AI предлагает — пользователь подтверждает»:

```
user: "Занеси трату 500р в пятёрочке сегодня"
  ↓
LLM tool_call propose_actual_transaction(amount_rub=500,
   description="пятёрочка", tx_date="сегодня")
  ↓
tool function (does NOT touch DB):
   ├─ amount_rub*100 → amount_cents
   ├─ "сегодня" → today.isoformat()
   ├─ EmbeddingService.suggest_category("пятёрочка") → Продукты 0.36
   └─ return {_proposal: true, kind_of: "actual", txn: {...}}
  ↓
route emits SSE "propose" event with payload
  ↓
frontend useAiConversation: setProposal(payload)
  ↓
AiProposalSheet → BottomSheet → ActualEditor with prefilled fields
  ↓
User edits if needed → "Сохранить" → standard POST /api/v1/actual
   (или Cancel → ничего не пишется)
```

Ключевое:
- AI **никогда не пишет в БД молча**.
- `category_hint` параметр выпилен — LLM выдумывал «Здоровье» для
  «кофе на работе». Сервер резолвит только по `description`,
  переданному VERBATIM.
- description передаётся **дословно** (фикс через system prompt:
  «pass user's description VERBATIM, e.g. 'Пятёрочка', not
  'покупка в магазине Пятёрочка'»).

### H. Forecast math (`ce2a3f9`)

`get_forecast` теперь возвращает готовые поля:
- `savings_capacity_monthly_cents` (=`forecast_balance_cents` clamped at 0)
- `savings_capacity_daily_cents` (=`monthly / total_days`)

System prompt правило (2): использовать эти поля **напрямую**, не
выводить save-rate из spend-rate. Числа теперь согласованы между
follow-up'ами:

| Q | A |
|---|---|
| Сколько я могу откладывать? | 209 338 ₽/мес или 6 753 ₽/день |
| Сколько это в месяц? | 209 338 ₽ |
| А в день? | 6 753 ₽ |

`209 338 / 31 ≈ 6 753` — согласовано.

## Threat flags

- **Token-cost upgrade** ×4 при переходе nano→mini. На пет-app
  при <100 chat-запросов/день — десятки центов в месяц,
  не нагрузка.
- **Synonyms hardcoded** — при добавлении новой категории
  пользователем (например «Котики»), embedding пойдёт от bare
  имени и recall будет ниже. Это не блокер: пороговая логика
  всё равно вернёт null при низкой confidence, и UI fallback'нется
  на ручной select.
- **Auto-seed in DEV** добавляет 7 sample транзакций. Это **DEV-only**
  (gated на `settings.DEV_MODE`), в проде не сработает.
