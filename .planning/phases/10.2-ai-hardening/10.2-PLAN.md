## Wave A — Infra wiring (предусловие для всего)

### A-1: проброс AI env vars в api
- `.env.example`: новая секция «AI / OpenAI» с placeholders + комментарий о ротации.
- `docker-compose.yml`: api environment проброс `OPENAI_API_KEY`, `LLM_PROVIDER`, `LLM_MODEL`, `AI_MAX_CONTEXT_MESSAGES`, `ENABLE_AI_CATEGORIZATION`. Только api — bot/worker не делают AI-вызовов.

### A-2: fail-fast валидатор
- `app/core/settings.py`: новое поле `SERVICE`. `validate_production_settings()`:
  - DEV-tier (BOT_TOKEN/INTERNAL_TOKEN/OWNER_TG_ID) релаксируется в DEV_MODE.
  - AI-tier (OPENAI_API_KEY) обязателен **всегда** для api/"" сервисов.
- Раскоментированный комментарий в коде даёт пользователю чёткое сообщение об ошибке.

### A-3: тестовая инфра
- `tests/conftest.py`: `os.environ.setdefault("OPENAI_API_KEY", "sk-test-fake-key-for-pytest-only")`.
- `docker-compose.test.yml`: тот же фейк-токен для api контейнера в integration-runner.

## Wave F — Write-flow

### F-19: propose tools
- `propose_actual_transaction(amount_rub, kind, description, tx_date)`.
- `propose_planned_transaction(amount_rub, kind, description, day_of_period)`.
- Резолв категории через `EmbeddingService.suggest_category(description)`.
- Парсинг даты: ISO / «сегодня» / «вчера» / today fallback.
- `**_ignored: Any` — старый `category_hint` ignored.
- Возврат `{_proposal: True, kind_of, txn: {...}}`.

### F-20: SSE 'propose' event в роуте
- В `_event_stream` после tool execution: если result содержит
  `_proposal=True`, эмитим `data: {"type":"propose","data":...}`.
- Тот же payload идёт в LLM как tool response (модель видит факты).

### F-21: frontend Proposal flow
- `types.ts`: `AiStreamEvent` discriminated union;
  `ProposalPayload = ActualProposalPayload | PlannedProposalPayload`.
- `useAiConversation`: state `proposal`, `dismissProposal()`.
- New component `AiProposalSheet`:
  - Fetches `listCategories()` + `listPeriods()`.
  - Renders `ActualEditor` или `PlanItemEditor` с prefilled.
  - onSave → `createActual` / `createPlanned`.
  - onCancel → just dismiss.
- `AiScreen` mounts `<AiProposalSheet />` поверх чата.

### F-22: SYSTEM_PROMPT routing
- Rule (4): на «занеси/добавь» — звать `propose_actual_*`,
  description передавать VERBATIM, категорию не угадывать.
- Rule (5): на «добавь в план» — `propose_planned_*`.
