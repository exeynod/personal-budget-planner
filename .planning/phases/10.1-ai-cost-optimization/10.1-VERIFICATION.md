## Human Verification

**Status: human_verify_pending** — requires live environment with `OPENAI_API_KEY`, running docker-compose stack, and a Telegram client.

### Steps

1. **Token reduction sanity (SC-1)**
   - Boot stack: `docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d`
   - Open Mini App, send the same chat probe twice (e.g. "сколько потратил в этом месяце?")
   - Compare `ai.usage` log lines from `docker compose logs api | grep ai.usage`
   - Expected: `prompt_tokens` lower than v0.3-pre-10.1 baseline (target: ≥50% reduction). If `prompt_tokens` ~ 200–300 on a fresh conversation it's working; pre-10.1 baseline was ~400–500.

2. **`/ai/usage` endpoint (SC-3)**
   - With a valid Telegram session, `curl -H "Authorization: tma <init-data>" http://localhost:8000/api/v1/ai/usage`
   - Expected: `{ "today": { "requests": ..., "prompt_tokens": ..., "est_cost_usd": ... }, "session_total": {...}, "buffer_size": N, "buffer_max": 1000 }`
   - Send a few chat messages, re-fetch — counters should grow.

3. **Embedding on category create (T-4.1)**
   - Create a new category named e.g. "Спорт" via UI Management screen.
   - In ActualEditor, type "тренировка" (≥3 chars).
   - Expected: AI suggestion box shows "Спорт" with confidence ≥ 0.5 (no longer requires renaming the category first).

4. **No-op rename skip (T-4.2)**
   - PATCH `/api/v1/categories/{id}` with `{"name": "Спорт"}` (same name as before).
   - Expected: api log shows neither `category.embedding.refreshed` nor `category.embedding.refresh_failed` for that category.id.
   - Then PATCH with a different name `{"name": "Фитнес"}` — embedding refresh log line should appear.

5. **Embedding LRU cache (T-4.3)**
   - Type the same description in ActualEditor twice (e.g. "кофе") with debounce → two `/ai/suggest-category?q=кофе` requests.
   - Expected: second request is faster (cached); no second `text-embedding-3-small` API call. Verify by absence of OpenAI request log on the second call.

6. **Rate limit (T-3.1)**
   - Send >10 chat messages within 60 s.
   - Expected: 11th request returns HTTP 429 (was previously 30/min).

7. **min_length=3 (T-3.2)**
   - `curl -H "Authorization: tma <init-data>" "http://localhost:8000/api/v1/ai/suggest-category?q=ab"` (2 chars)
   - Expected: HTTP 422 with FastAPI validation error mentioning min_length.
