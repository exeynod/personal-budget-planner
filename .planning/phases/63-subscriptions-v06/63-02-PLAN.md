| Boundary | Description |
|----------|-------------|
| User swipe → post mutation | post создаёт транзакцию-списание; swipe легко срабатывает случайно → нужен confirmationDialog |
| Editor save → API | create (legacy) + follow-up PATCH (V10) — два запроса; частичный сбой возможен |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-01 | Tampering/DoS | swipe post double-trigger | mitigate | confirmationDialog (явное подтверждение) + VM submitting guard; повторный post — no-op in-flight |
| T-63-02 | Information Disclosure | editor/row error surfaces | mitigate | banner читает viewModel.mutationError (фиксированная RU); 0 error.localizedDescription в save/post catch |
| T-63-05 | Repudiation | случайный post/unpost через swipe | mitigate | confirmationDialog с titleVisibility .visible перед денежной мутацией |
| T-63-06 | Consistency | create + follow-up PATCH частичный сбой (sub создан, day/account не записан) | accept | create успешен → подписка существует; PATCH-сбой → mutationError + load() показывает реальное состояние; пользователь может переоткрыть editor. Документировать в SUMMARY. |
