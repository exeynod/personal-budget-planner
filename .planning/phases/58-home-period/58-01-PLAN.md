# Plan 58-01: Home empty state — нейтральная коррекция

**Phase:** 58 — Home & Period (v06 native)
**Status:** Shipped
**Date:** 2026-05-11

## Goal

Убрать вводящий в заблуждение текст «Завершите onboarding» из v06 HomeView empty state. AppRouter гарантирует `is_onboarded=true` до отрисовки `HomeView`, поэтому правильная причина пустого стейта — отсутствие активного периода (worker close_period_job ещё не создал next period, или dev-сид зачистил данные).

## Changes

### `ios/BudgetPlanner/Features/Home/HomeView.swift`

`.noActivePeriod` case:

**Было:**
```swift
ContentUnavailableView(
    "Нет активного периода",
    systemImage: "calendar.badge.exclamationmark",
    description: Text("Завершите onboarding, чтобы создать первый период.")
)
```

**Стало:**
```swift
ContentUnavailableView {
    Label("Период ещё не открыт", systemImage: "calendar.badge.clock")
} description: {
    Text("Новый месячный период создаётся автоматически после закрытия предыдущего или при первой трате. Добавьте операцию через «+» вверху или обновите экран.")
} actions: {
    Button("Добавить трату") { showingEditor = true }
        .buttonStyle(.borderedProminent)
    Button("Обновить") { Task { await viewModel.load() } }
        .buttonStyle(.bordered)
}
```

Иконка изменена с `.exclamationmark` (тревога) на `.clock` (нейтральное ожидание). Action wiring: primary жмёт `showingEditor = true` — открывает existing TransactionEditor sheet; secondary re-fetch.
