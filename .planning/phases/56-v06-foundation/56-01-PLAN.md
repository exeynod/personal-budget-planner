# Plan 56-01: Theme Toggle Foundation

**Phase:** 56 — v06 Native Rebuild — Foundation
**Status:** Shipped
**Date:** 2026-05-11

## Goal

Сделать `ui.theme` тумблер рабочим в обе стороны: из `MainShell` (v06) → `V10MainShell`, из `V10MainShell` (через `ThemePickerSheet`) → `MainShell`. До этой фазы AppRouter принудительно фолбэкал `"v06"` в `"v10"` через self-heal, и в `ThemePickerSheet` не было опции «v06».

## Changes

### `ios/BudgetPlanner/App/AppRouter.swift`

- Default `@AppStorage("ui.theme")` → `Theme.maximalPoster.rawValue` (раньше `"v10"`).
- Удалена computed `theme` с whitelist `["v06", "v10"]`.
- Условие переключения: `themeRaw == "v06"` → `MainShell`, иначе → `V10MainShell`.
- Удалена self-heal перезапись `themeRaw = "v10"` в `.task`.

### `ios/BudgetPlanner/Features/Management/SettingsView.swift`

- Добавлен `@AppStorage("ui.theme")` со значением `Theme.maximalPoster.rawValue`.
- Добавлена секция `designSection` после `aiSpendSection`: Section «Дизайн» с Button(LabeledContent) «Переключить на V10 → MAXIMAL POSTER». Запись `themeRaw = Theme.maximalPoster.rawValue` инициирует переход в `V10MainShell` через `AppRouter` body re-eval.

### `ios/BudgetPlanner/FeaturesV10/Management/ThemePickerSheet.swift`

- Содержимое обёрнуто в `ScrollView` (4 опции не помещаются в фиксированный sheet под таб-баром).
- Добавлен legacy-row под `ForEach(themes)`: Button «СТАРЫЙ IOS» c домик-иконкой, который пишет `themeRaw = "v06"` и закрывает sheet.
- Helper'ы `legacyV06Value: String { "v06" }` и `legacySwatch: some View`.

## Verification

4. Plist `com.exeynod.BudgetPlanner.plist`: `ui.theme` корректно обновляется на каждом переключении.
