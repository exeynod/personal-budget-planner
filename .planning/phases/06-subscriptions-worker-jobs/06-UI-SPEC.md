## Spacing Scale

Declared values — canonical 8-point scale only:

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, dot/pill internal padding |
| `--space-2` | 8px | Inline element spacing, badge padding, compact card internal gaps |
| `--space-4` | 16px | Default screen horizontal padding, card padding, row vertical rhythm |
| `--space-6` | 24px | Section-level gap between hero/timeline/list |
| `--space-8` | 32px | Empty-state vertical padding |
| `--space-12` | 48px | Bottom padding (accounts for MainButton + safe area) |

Structural exceptions (not spacing scale entries — CSS-drawn geometry values):
- Timeline track height: 4px (structural)
- Timeline dot diameter: 12px (structural, hit-target geometry)
- Today-line height: full track container height (100% relative)
- Touch targets (toggle, active/inactive buttons): minimum 44px height — matches existing `tabBar` height convention from `HomeScreen.module.css`

Note: `--space-3` (12px) and `--space-5` (20px) exist in `tokens.css` as structural values. They are NOT part of the declared spacing scale for this phase. Usages previously at 12px (compact card internal gaps) use `--space-2` (8px); usages previously at 20px (row vertical rhythm) use `--space-4` (16px).

Source: `frontend/src/styles/tokens.css` (spacing), `HomeScreen.module.css` (44px tab convention).

## Typography

All sizes and weights drawn directly from existing `tokens.css`. Maximum 4 sizes, 2 weights.

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| Body | 15px | 400 (regular) | 1.5 | `--text-base`, `--weight-regular` |
| Label / pill / cycle badge | 13px | 400 (regular) | 1.4 | `--text-sm`, `--weight-regular` |
| Heading (hero values) | 20px | 600 (semibold) | 1.2 | `--text-lg`, `--weight-semibold` |
| Display (hero amount) | 32px | 600 (semibold) | 1.1 | `--text-2xl`, `--weight-semibold` |

Usage rules:
- Hero block "monthly load" amount: `--text-2xl` + `--weight-semibold`
- Hero block label ("Нагрузка в месяц"): `--text-sm` + `--weight-regular` + `--color-text-muted`
- Subscription card name: `--text-base` + `--weight-semibold`
- "через N дн." pill: `--text-sm` + `--weight-regular`
- Cycle badge "[мес]" / "[год]": `--text-sm` + `--weight-regular` + `--color-text-dim`
- Section headers (if any): `--text-sm` + `--weight-semibold` + `--color-text-muted` (uppercase tracking — matches existing section header pattern)
- Editor field labels: `--text-sm` + `--weight-regular` + `--color-text-muted`

Note: `--weight-medium` (500) and `--weight-bold` (700) exist in `tokens.css` but are not used in Phase 6. Medium usages collapse to `--weight-regular` (400); bold usages collapse to `--weight-semibold` (600).

Source: `frontend/src/styles/tokens.css` (all typographic vars).

## Color

All values from existing `tokens.css`. No new color variables introduced.

| Role | Token | Hex (dark theme) | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `--color-bg` | `#0e1116` | Screen background, full-bleed areas |
| Secondary (30%) | `--color-surface` / `--color-surface-2` | `#1c2230` / `#232a3a` | Subscription cards, editor sheet, hero block |
| Accent (10%) | `--color-primary` | `#4ea4ff` | Today-line on timeline, active toggle, primary form buttons |
| Destructive | `--color-danger` | `#ff5d5d` | Delete button in SubscriptionEditor, ≤2 days charge warning dot/pill |

Accent reserved for:
1. Today-line vertical marker on the timeline track
2. Toggle `is_active` — ON state indicator
3. "Добавить подписку" MainButton background (via Telegram SDK MainButton, follows primary color)
4. Focused/active state on form inputs in SubscriptionEditor (border-color)

Semantic colors for timeline and pills (pre-existing tokens, not accent):
- `--color-danger` (`#ff5d5d`): dot + pill when `daysUntilCharge <= 2`
- `--color-warn` (`#ffb547`): dot + pill when `daysUntilCharge <= 7`
- `--color-text-dim` (`#5d6577`): dot + pill when `daysUntilCharge > 7`
- `--color-success` (`#2ecc71`): `is_active = false` badge / inactive subscription label
- `--color-surface-2` (`#232a3a`): inactive subscription card background (dimmed)
- `--color-border` (`#2a3142`): timeline track base line, card borders

Soft variants for backgrounds:
- `--color-danger-soft` (`rgba(255,93,93,0.14)`): pill background when ≤2 days
- `--color-success-soft` (`rgba(46,204,113,0.14)`): pill background for inactive badge

Source: `frontend/src/styles/tokens.css`.

## Component Inventory

New components introduced in Phase 6 (follow existing CSS Modules + BEM-ish naming):

### `SubscriptionsScreen.tsx` + `SubscriptionsScreen.module.css`
Layout: `.root` → max-width 375px, padding `var(--space-4)` horizontal, `var(--space-12)` bottom, `var(--color-bg)` background. Identical shell to `HomeScreen.tsx`.

Sections (top to bottom):
1. **Hero block** (inline in screen, no sub-component): two stat rows inside a `--gradient-hero` card. Rows: «Активных подписок: N» and «Нагрузка в месяц: X ₽». Use `SectionCard` component wrapper.
2. **SubscriptionTimeline** card: `SectionCard` wrapper. Title «Этот месяц». CSS-only horizontal track (height 4px, `--color-border` background). Dots (`12px × 12px`, `border-radius: 50%`) positioned `left: {pct}%`. Today-line (`2px solid --color-primary`, full height of container). Tooltip: native `title` attribute on dot element (lightweight, no portal).
3. **Subscription list**: flat list sorted `next_charge_date ASC`. Each card is a `<button>` row (tap → open edit sheet). Structure per card: [Name + cycle badge] [amount] [days-pill] [active toggle].

### `SubscriptionEditor.tsx` + `SubscriptionEditor.module.css`
Pattern: identical props contract to `ActualEditor.tsx` — `initial`, `onSave`, `onDelete?`, `onCancel`. Rendered inside existing `BottomSheet.tsx`.

Fields in order:
1. `name` — text input, placeholder «Название подписки»
2. `amount` — number input (рубли, convert ↔ копейки via `parseRublesToKopecks` / `formatKopecksToRubles` — reuse from `ActualEditor`)
3. `cycle` — segmented control (two buttons «Мес» / «Год»), height 44px
4. `next_charge_date` — `<input type="date">`, value ISO string
5. `category` — `<select>`, filtered to active categories only
6. `notify_days_before` — `<input type="number" min="0" max="30">`, default from Settings
7. `is_active` — toggle (CSS-only, same visual as existing toggles in CategoriesScreen)

Delete button (edit mode only): `--color-danger` text, bottom of sheet, same pattern as ActualEditor.

### Navigation addition (`App.tsx`)
Add «Подписки» tab to bottom nav bar. Tab label: «Подписки». Nav icon: text emoji «🔁» or plain text — follow existing nav pattern (existing tabs use text labels, no icon library).

### Settings extension (`SettingsScreen.tsx`)
Add one new field row after `cycle_start_day`:
- Label: «Уведомление за (дней)»
- Input: `<input type="number" min="0" max="30">`
- Save behaviour: same `PATCH /settings` debounce/on-blur pattern as existing `cycle_start_day` field

## Interaction States

### SubscriptionTimeline
| State | Visual |
|-------|--------|
| Default | Dots on track, today-line in primary, no selection |
| Dot hover/tap | Native `title` tooltip appears (no JS tooltip portal) |
| Empty month (no subs) | Timeline hidden; empty state shown instead |

### Subscription card row
| State | Visual |
|-------|--------|
| Default | Full opacity |
| Inactive (`is_active=false`) | Card background `--color-surface-2`, name text `--color-text-muted`, «неактивна» badge |
| Tap | Opens SubscriptionEditor BottomSheet (same `BottomSheet.tsx` component) |
| Toggle tap (is_active) | Immediate optimistic update + PATCH; revert on error + toast |

### SubscriptionEditor
| State | Visual |
|-------|--------|
| Create mode | No delete button, title «Новая подписка» |
| Edit mode | Delete button visible (danger color), title «Редактировать подписку» |
| Saving (busy) | Save button disabled + spinner text (matches ActualEditor busy guard pattern) |
| Validation error | Inline error below field, `--color-danger` text, `--text-sm` |

### Edge cases
| Case | Behaviour |
|------|-----------|
| No `tg_chat_id` set (bot not linked) | Worker job skips silently (backend). UI shows no indication — no user-facing state needed. |
| `charge-now` on already-charged date | HTTP 409 → toast «Подписка уже списана за этот день» |
| Archived category in subscription | Archived category still shown in editor (subscription pre-exists) but `<select>` marks it «{name} (архив)» |
