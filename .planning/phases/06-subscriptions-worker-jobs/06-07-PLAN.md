  <name>Task 2: Ручной trigger worker job'ов + проверка эффектов</name>
  <what-built>Два worker job'а notify/charge с advisory lock и idempotency.</what-built>
  <how-to-verify>
    1. **Подготовка**: создать через UI тестовую подписку с next_charge_date = сегодня (для charge) и через 2 дня (для notify, если notify_days_before=2).

    2. **charge_subscriptions_job manual trigger**:
       ```bash
       docker-compose exec worker python -c "import asyncio; from app.worker.jobs.charge_subscriptions import charge_subscriptions_job; asyncio.run(charge_subscriptions_job())"
       ```
       Проверить:
       - В логах сообщение `charge_subscriptions: 1 subs due today` и `sub_id=X charged, next=YYYY-MM-DD`
       - В БД: `SELECT * FROM planned_transaction WHERE source='subscription_auto' ORDER BY id DESC LIMIT 1` — новая строка
       - В UI экран План текущего периода — новая строка с маркером (если он реализован — sketch 005-B)
       - У подписки в БД: `next_charge_date` сдвинут на +1 month

    3. **Idempotency check**: запустить тот же job ВТОРОЙ раз сразу после первого:
       - В логах: либо `nothing to notify` (если next_charge_date уже сдвинут), либо `already charged today, skip`
       - Никаких новых planned-строк в БД

    4. **notify_subscriptions_job manual trigger**:
       ```bash
       docker-compose exec worker python -c "import asyncio; from app.worker.jobs.notify_subscriptions import notify_subscriptions_job; asyncio.run(notify_subscriptions_job())"
       ```
       Проверить:
       - Push пришёл в Telegram-чат с правильным текстом «🔔 Подписка «{name}»...»
       - В логах `notify_subscriptions: sent for sub_id=X`

    5. **POST /subscriptions/{id}/charge-now**: через curl или Mini App вызвать ручное списание:
       ```bash
       curl -X POST "https://localhost/api/v1/subscriptions/{id}/charge-now" -H "X-Telegram-Init-Data: ..."
       ```
       Должен вернуть 200 + `{planned_id, next_charge_date}`. Повторный вызов в тот же день → 409.

    6. **SET-02 only-new behaviour**: в Settings изменить notify_days_before с 2 на 7. Создать НОВУЮ подписку без явного notify_days_before — должна получить 7. У существующих подписок значение не меняется.
  </how-to-verify>
  <resume-signal>Type "approved" если все 6 шагов работают, иначе опиши.</resume-signal>
