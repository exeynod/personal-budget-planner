<objective>

Pro-gated endpoint `GET /api/v1/tax/reserve`. Возвращает рекомендуемый
резерв = `business_income * rate * 1.05` (5% safety margin).
</objective>

## Task

1. Service `app/services/tax_reserve.py`:
   - `round_to_cent(decimal_rub) -> int` — HALF_UP rounding в копейки.
   - `RATE_BY_REGIME = {'nalog_4': Decimal('0.04'), 'nalog_6': Decimal('0.06')}`.
   - `calculate_tax_reserve(db, user_id, period_start, period_end, regime)`
     → `TaxReserveResult` dataclass (period_*, income_cents,
     business_income_cents, regime, tax_owed_cents, reserve_recommended_cents).
2. Endpoint `GET /api/v1/tax/reserve` (`app/api/routes/tax.py`) — Pro-gated
   через `require_pro`; query params `period_start`, `period_end`, `regime`
   (Literal `nalog_4` | `nalog_6`, default `nalog_4`).
3. Tests `tests/test_tax_reserve.py` — 4 scenarios:
   - `round_to_cent` (basic + HALF_UP edge .995).
   - Rate constants exact Decimal.
   - DB-backed: zero income → all zeros.
   - DB-backed: mixed 50K business + 30K personal → tax 2000₽, reserve 2100₽.
