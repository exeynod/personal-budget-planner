## Decisions Made

| # | Decision | Rationale |
| --- | --- | --- |
| 1 | Replace legacy `onboarding_router` mount with `onboarding_v10_router` (not parallel-mount with prefix) | CONTEXT D-04 explicitly drops v0.x backwards compat. Single user, single iOS client (already being rewritten for v1.0). Coexisting routes would create path collision. |
| 2 | Stub `templates_router` with 410 Gone instead of deleting the file | `app.api.router` imports `templates_router` at module top-level. Removing the import breaks the public_router build chain. Stub keeps boot stable while signalling deprecation to legacy clients. |
| 3 | Co-locate `DepositResponse` in `routes/savings.py` instead of `schemas/savings.py` | Plan suggested reusing `ActualRead` from legacy schemas. That model is tightly coupled to v0.x `ActualTransaction` shape; the v1.0 deposit response only needs 6 fields. Co-location avoids upstream-schema churn. |
| 4 | Pre-validate `account_id` in `savings.create_deposit` (service-layer) | Without pre-check, missing `account_id` triggers `IntegrityError` from FK constraint after the deposit insert — caller sees a 500 instead of a clean 404. Service-layer pre-check is the cleanest fix per Rule 1 deviation. |
| 5 | Add `mode='before'` ISO-date validator to `due` field instead of dropping `strict=True` from goal schemas | Preserves T-22-12-01 (no implicit type coercion) for non-date fields while making the wire JSON contract work. Two-step validation (coerce string → date in `before`, future-check on the parsed `date`) keeps validators composable. |

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Pydantic strict=True rejected ISO date strings on goal.due**
- **Found during:** Task 8 (integration tests, first goals POST).
- **Issue:** `GoalCreate.due` typed as `Optional[date]` with `strict=True` rejected the JSON wire format string with 422 (`Input should be a valid date`).
- **Fix:** Added `_coerce_iso_date` helper + `mode='before'` field validators on `GoalCreate.due` / `GoalUpdate.due` / `OnboardingGoalItem.due`. The native `_ensure_future_date` validator runs after coercion.
- **Files modified:** `app/api/schemas/goals.py`, `app/api/schemas/onboarding_v10.py`.
- **Verification:** `tests/api/test_goals_api.py::test_create_goal_happy` and `test_complete_v10_with_goal_and_config` pass with ISO date strings on the wire.
- **Committed in:** `b3ef541`.

**2. [Rule 1 — Bug] Missing-account on deposit returned 500 instead of 404**
- **Found during:** Task 8 (`tests/api/test_savings_api.py::test_post_deposit_missing_account_404`).
- **Issue:** `savings.create_deposit` inserted the `actual_transaction` row first; bad `account_id` then triggered `ForeignKeyViolationError` (asyncpg) → SQLAlchemy `IntegrityError` → FastAPI 500. The route's `except (GoalNotFoundError, AccountNotFoundError)` clause was unreachable.
- **Fix:** Added `from app.services.accounts import get_or_404 as get_account_or_404` + a pre-flight call in `create_deposit` so missing accounts surface `AccountNotFoundError` cleanly. Local import to avoid circular dep.
- **Files modified:** `app/services/savings.py`.
- **Verification:** `tests/api/test_savings_api.py::test_post_deposit_missing_account_404` returns 404.
- **Committed in:** `b3ef541`.

**3. [Rule 3 — Blocking] PlanTemplateItem cleanup expanded beyond plan scope**
- **Found during:** Task 1 (initial cleanup sweep).
- **Issue:** Plan listed `app/services/templates.py` and `app/api/routes/templates.py` for cleanup. Reading the codebase surfaced two more blockers:
  - `tests/helpers/seed.py` had a module-level `from app.db.models import PlanTemplateItem` — would break ALL tests that import the seed helper (transitively many tests).
  - `tests/conftest.py` truncate-set strings include `plan_template_item` — `TRUNCATE` would fail with "relation does not exist" on post-22 DB.
- **Fix:** Extended cleanup to those two files. `seed_plan_template_item` now raises `NotImplementedError` with migration guidance. Truncate sets refreshed: drop `plan_template_item`, add `account / goal / savings_config`.
- **Files modified:** `tests/helpers/seed.py`, `tests/conftest.py`.
- **Verification:** New tests under `tests/api/` pass (truncate works, seed helper imports cleanly).
- **Committed in:** `c31863d`.
