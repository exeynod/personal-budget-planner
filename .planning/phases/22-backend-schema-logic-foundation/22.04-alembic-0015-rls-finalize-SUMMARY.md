# Phase 22 Plan 04: Alembic 0015 RLS Finalize Summary

Финальная Phase 22 schema-миграция: RLS-policies на новых v1.0 таблицах (goal, savings_config) + composite FK на actual_transaction.parent_txn_id для cross-tenant защиты (BE-16). Forward+back+forward verified e2e в local docker compose.

## What Was Built

**`alembic/versions/0015_v10_rls_finalize.py`** — 4 шага в `upgrade()`, симметричный `downgrade()`:

1. **RLS на `goal`:** ENABLE+FORCE ROW LEVEL SECURITY (idempotent re 0014) + CREATE POLICY `tenant_isolation_goal` с `coalesce(NULLIF(current_setting('app.current_user_id', true), '')::bigint, -1)` defense pattern из 0006.
2. **RLS на `savings_config`:** идентично goal — `tenant_isolation_savings_config`.
3. **Idempotent GRANTs** на goal/savings_config + goal_id_seq к `budget_app` role (defense-in-depth).
4. **Composite FK на actual_transaction.parent_txn_id:**
   - `op.drop_constraint("fk_actual_parent_txn", ...)` — drop simple FK из 0014.
   - `op.create_unique_constraint("ux_actual_id_user", "actual_transaction", ["id", "user_id"])` — composite UNIQUE как FK target (PG требование).
   - Raw SQL `ADD CONSTRAINT fk_actual_parent_txn_composite FOREIGN KEY (parent_txn_id, user_id) REFERENCES actual_transaction (id, user_id) ON DELETE CASCADE` — `op.create_foreign_key()` не поддерживает composite FK на non-PK target, паттерн идентичен 0013 для category.parent_id.

`down_revision = "0014_v10_actual_goal_savings"` (правильный chain). Файл 193 строки с paranoid commentary на каждом шаге.

## Decisions Made

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | ENABLE+FORCE ROW LEVEL SECURITY дублируем здесь | 0014 уже выставил, но idempotent в PG; paranoid против manual disable между миграциями (no penalty, defense-in-depth) |
| 2 | Idempotent GRANTs повторены | 0014 уже granted; defense-in-depth re manual revoke; PG GRANT no-op если уже granted |
| 3 | REVOKEs в downgrade включены | Симметрия + документация intent; реально privileges испаряются с DROP TABLE в 0014 downgrade |
| 4 | Composite FK через raw SQL `op.execute()` | `op.create_foreign_key()` API не поддерживает composite FK на non-PK target; паттерн идентичен 0013 |
| 5 | Drop simple FK + ADD composite (а не ALTER) | PG не имеет `ALTER CONSTRAINT` для смены columns в FK; drop-then-add — единственный путь |
| 6 | Naming `tenant_isolation_<table>` (не `<table>_user_isolation`) | CONTEXT D-08 — единая convention для v1.0 (account/goal/savings_config); legacy 0006 naming зарезервирован для original 9 таблиц |
| 7 | Восстановление simple FK в downgrade | Чтобы 0014 был "snapshot-clean" — после `downgrade -1` schema идентична post-0014 state, для clean re-test 0014 standalone |

## Deviations from Plan

- **Found during:** Task 1 — verification против acceptance criteria

- **Files modified:** `alembic/versions/0015_v10_rls_finalize.py`
