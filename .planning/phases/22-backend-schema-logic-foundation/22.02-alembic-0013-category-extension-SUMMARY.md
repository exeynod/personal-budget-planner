## Decisions Made

- **Composite FK через raw DDL** — SQLAlchemy `op.create_foreign_key` не делает self-reference composite to-same-table; `op.execute("ALTER TABLE ... ADD CONSTRAINT ...")` чище чем `ForeignKeyConstraint` в `__table_args__`.
- **Inline transliteration map (no unidecode dep)** — миграция self-contained, deterministic на любом env. Покрывает существующий seed (Продукты, Кафе, Транспорт, Развлечения, Здоровье, Спорт, Книги, Подарки, Сервисы, Зарплата) + любой cyrillic input.
- **`code` truncate to 40 chars** — защита от потенциально длинных имён + collision suffix.
- **`ord` clamp `least(sort_order, 99)`** — текущий dev seed имеет `sort_order` до 80 (=`'80'`), но защитный clamp на случай legacy данных с >99.
- **Partial unique `uq_category_user_code` WHERE NOT is_archived** — соответствует существующему паттерну (0010 `uq_category_user_id_name`); архивные категории могут переиспользовать code.
- **Downgrade re-includes RLS + GRANTs для plan_template_item** — иначе после round-trip таблица была бы без защиты, что критично если кто-то остановится на 0012 в dev.
