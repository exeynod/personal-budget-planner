# Phase 22 Plan 01: Alembic 0012 — User.income_cents + Account + RLS Summary

Single-file alembic migration `0012_v10_user_account.py` adds `app_user.income_cents` BIGINT NULL column and creates the new multi-tenant `account` table with FK to `app_user`, hardened RLS policy `tenant_isolation_account`, partial-unique index enforcing one primary account per user, and explicit GRANTs to `budget_app`.

### Indexes

- `account_pkey` — PK on `id`
- `ix_account_user_id` — btree on `user_id` (tenant query path)
- `ix_account_user_primary_one` — UNIQUE btree on `user_id` WHERE `"primary" = true` (T-22-01-02)

### Policies

- `tenant_isolation_account` — USING + WITH CHECK `(user_id = coalesce(NULLIF(current_setting('app.current_user_id', true), '')::bigint, -1))`. ENABLE ROW LEVEL SECURITY + FORCE ROW LEVEL SECURITY.

### GRANTs

- `GRANT SELECT, INSERT, UPDATE, DELETE ON TABLE account TO budget_app`
- `GRANT USAGE, SELECT ON SEQUENCE account_id_seq TO budget_app`
