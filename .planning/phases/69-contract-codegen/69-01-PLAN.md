<objective>
**B1 — Backend: clean OpenAPI + deterministic dump target.** The OpenAPI schema is
the single source of truth for the whole phase (B2 web + B3 iOS codegen consume it).
Two jobs:

1. **response_model audit.** Audit every public route in the 8 in-scope domains
   (subscriptions, categories, actuals, me, ai, accounts, savings, goals) for an
   explicit `response_model=`. The fix is: **type the read-DTO routes; document +
   exempt the free-form dump/compliance routes.** Forcing a precise Pydantic model
   onto an arbitrary nested data-dump risks reshaping its keys (a regression), so
   those routes stay `response_model=None` and the contract-coverage check EXEMPTS
   them by an explicit allowlist. Confirmed gaps from grep:
   - `app/api/routes/me.py` — 5 decorators, only 1 (`PATCH /me`) carries
     `response_model`. The consent-set (`:106`) and consent-delete (`:141`) handlers
     return a structured read shape (e.g. `{"pdn_consent_at": ..., "revoked": bool}`)
     → these DO get typed Pydantic response models (in `me_v10.py`). The
     **data-export (`:170`) handler returns an arbitrary nested export structure —
     leave it `response_model=None` / documented dict and EXEMPT it** (do NOT
     synthesize a model — reshaping the export keys is a regression). The
     **account-delete (`:202`) handler: IF its body is a structured fixed shape,
     type it; IF it returns a free-form dump, leave response_model=None and exempt
     it** (decide per the actual return at exec time and document the call). There
     is NO `GET /me` route in this module — `PATCH /me` already uses `MeV10Response`
     (Phase 67); the legacy GET lives in `app.api.router` and `build_me_response`
     (me.py:46) is a helper, not a route. Leave them.
   - `app/api/routes/ai.py` — the SSE `/ai/chat` stream is legitimately not a JSON
     `response_model` (StreamingResponse); the `_agg` helper at `:718` is an
     internal aggregator, NOT a route return — confirm it is not a route response.
     `/ai/history` + `/ai/conversation` already carry response_model. Only add
     models where a route returns a bare dict on the JSON read surface.
   - subscriptions/categories/actual/accounts/savings/goals: grep shows
     decorators == response_model (covered). Re-verify each file; if a `-> dict`
     route exists on the public surface, type it (if read-DTO) or exempt it (if a
     documented free-form dump).

2. **Deterministic dump.** Add `contract/dump_openapi.py` that imports
   `from main_api import app` (repo-root entry; `app/main_api.py` re-exports it)
   and writes `app.openapi()` to `contract/openapi.json` with
   `json.dump(..., indent=2, sort_keys=True, ensure_ascii=False)` + a trailing
   newline. Sorted keys make the artifact stable for the B5 git-diff sync-guard.
   Add a `make contract` target running it inside the docker `api` container.

Purpose: a clean, deterministic, regenerable OpenAPI artifact that B2/B3 codegen
build on. This is the foundation wave — nothing else can start until openapi.json
exists.
Output: typed me read-DTO responses (consent), documented+exempt export/delete dump
routes, contract/dump_openapi.py, contract/openapi.json, make contract target, a
contract guard test. Likely NO alembic migration (types, not DB schema).
</objective>
