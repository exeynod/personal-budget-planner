<objective>
**B2 — Web codegen (openapi-typescript).** Generate TypeScript types from the
Phase-69-B1 `contract/openapi.json` artifact, establishing the generated schema as
the source of truth for web DTOs. This plan ONLY generates + diffs — it does NOT
yet migrate consumers (that is 69-04, read-DTO first). Keeping generation and
migration in separate plans keeps each within budget and isolates the
zero-regression gate.

Work:
1. `cd frontend && npm i -D openapi-typescript`.
2. Add a `gen:api` script: `openapi-typescript ../contract/openapi.json -o src/api/generated/schema.ts`.
3. Generate `frontend/src/api/generated/schema.ts` (committed). openapi-typescript
   emits `paths` + `components["schemas"]` namespaces — leave that shape as-is; the
   adapter mapping onto current call-sites is 69-04's job (Claude's discretion per
   CONTEXT).
4. Produce `drift-report.md`: diff the generated `components["schemas"]` against the
   handwritten `frontend/src/api/types.ts`, focusing on `CategoryV10`. The B1 audit
   confirmed `CategoryRead` now emits the v1.0 fields. Be precise about
   required-vs-optional: **`code` and `ord` are required-on-wire (no server default)
   → generated as non-optional; `plan_cents` (=0), `rollover` (="misc"), `paused`
   (=False), `parent_id` (=None), `tag` (="personal") carry server defaults → NOT in
   the OpenAPI `required` set → generated as OPTIONAL.** Additionally call out the
   `tag` field explicitly: `CategoryRead` emits `tag: CategoryTagStr`
   ("personal"|"business"|"mixed", Phase 36) on the wire, but the handwritten
   `CategoryV10` DTO LACKS it entirely — flag this as a missing-field drift so 69-04
   adds it. Document each drift so 69-04 can remove the stubs / add `tag` safely.
   Catch any other name/nullability mismatches.
5. Idempotency: running `gen:api` twice yields a byte-identical schema.ts.

Purpose: a generated, source-of-truth web type set + a drift report that makes the
69-04 migration mechanical and safe. No consumer migration here.
Output: openapi-typescript devDep, gen:api script, generated/schema.ts, README +
drift-report. Build/typecheck:test/vitest stay green (no consumer changes yet).
</objective>
