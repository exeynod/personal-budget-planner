# ПЛАН 1 — Устранение косяков (remediation)

Приоритеты: **P0** = ship-blocking, **P1** = корректность/безопасность, **P2** = edge/minor.

## P0 — Blockers

### P0-1 · Backend не отдаёт V10-поля подписок (BE-F1) — BLOCKER
- **Симптом:** iOS phase 63 пишет `day_of_month/account_id`, но при reload получает `nil`; posted-бейдж (`posted_txn_id`) никогда не показывается. Фаза 63 нерабочая end-to-end.
- **Фикс:** в `app/api/routes/subscriptions.py` сменить `response_model=SubscriptionRead` → `SubscriptionReadV10` для list (46), post (56), patch (104). Убедиться, что ORM→schema проносит 3 поля.
- **Проверка:** integration-тест: создать подписку с day/account → GET → поля присутствуют; post → posted_txn_id в ответе.
- **Файлы:** `app/api/routes/subscriptions.py`, `app/api/schemas/subscriptions.py`.

### P0-2 · Web-сборка сломана (FE-F1, FE-F2, FE-F3) — BLOCKER
- **Симптом:** `npm run build` (`tsc -b && vite build`) падает (exit 2). `vite build` сам по себе зелёный — маскирует.
- **Фиксы:**
  - `frontend/src/api/v10/analytics.ts:19` — `AnalyticsRange` импортировать из `'../analytics'` (не из `'../types'`).
  - `frontend/src/screensV10/Ai/AiView.tsx:182` — тип `bottomRef` → `HTMLLIElement` (или anchor `<div>`).
  - CI: убедиться, что пайплайн гоняет `npm run build` (с tsc-гейтом), а не голый `vite build`; почистить tsc-ошибки в тестах.

### P0-3 · `suppressForbiddenHandler` — неверная посылка + ослабление auth (iOS phase 64 / ARCH-A4) — BLOCKER-class
- **Симптом:** `require_pro`=402, а флаг подавляет 403. Non-pro и так получает 402 → ловится `catch`→`nil` (подсказка скрыта) БЕЗ участия флага. Флаг же глушит настоящий 403 (auth-fail) на AI-пути → токен-сбой там не разлогинит.
- **Фикс:** удалить `suppressForbiddenHandler` из `APIClient` (вернуть строгий 403→`onUnauthenticated`); в `AISuggestCategoryAPI.suggest` полагаться на существующий nil-on-error (402 уже падает в `serverError`→catch→nil). Опционально: маппить 402 в типизированный `.proRequired` без logout.
- **Проверка:** тест P1-7 ниже.
- **Файлы:** `ios/.../Networking/APIClient.swift`, `Endpoints/AISuggestCategoryAPI.swift`.

## P1 — Major (корректность / безопасность)

### P1-1 · Эмбеддинги категорий не создаются после онбординга (BE-F2)
- `_refresh_embedding` зовёт `upsert_category_embedding` без kw-only `user_id` → `TypeError`, проглатывается; фоновая сессия без `set_tenant_scope`. → suggest-category для пользовательских категорий молча не работает (только substring-fallback).
- **Фикс:** прокинуть `user_id` в `_refresh_embedding` + `set_tenant_scope(session, user_id)` в фоновой сессии. Тест на post-onboarding категорию.
- **Файлы:** `app/api/routes/categories.py:87,107,163`.

### P1-2 · Double-post race на `/subscriptions/{id}/post` (BE-F4)
- Идемпотентность только по in-memory `posted_txn_id is None`, без `FOR UPDATE`/unique-constraint. Два параллельных post → две ActualTransaction + двойная balance-дельта + осиротевшая txn.
- **Фикс:** `SELECT … FOR UPDATE` по подписке в начале `post_subscription`, либо partial unique index `posted_txn_id WHERE NOT NULL` + IntegrityError→409.
- **Файлы:** `app/services/subscriptions.py:390-431`, миграция (index).

### P1-3 · Утечка `error.localizedDescription` в UI (iOS-F1, iOS-F2)
- `TransactionEditor.save()` (`:361`) и `SettingsViewModel.load/save` (`:34,58`) пишут сырой error в баннер — утечка + нарушение собственной v06-конвенции фиксированных RU-копий.
- **Фикс:** фиксированная RU-копия, raw → `print` под `#if DEBUG`. Кластер: Template/Categories/Analytics/CategoryDetail/Home/AIChat — те же (см. R1-error-mapper).
- **Файлы:** `TransactionEditor.swift`, `SettingsView.swift`, + кластер.

### P1-4 · Денежные мутации Savings без seam + без тестов (iOS-F3, QA-F1, QA-F2)
- `SavingsViewModel`/`GoalDetailViewModel` бьют по static API без инъецируемого seam → `deposit`/`createGoal`/`deleteGoal`/optimistic-revert НЕ покрыты поведенчески (deposit — реальные деньги). Плюс нет `reloadPending`-коалесинга (есть только в Subscriptions) → устаревший снапшот после депозита при активном pull-to-refresh.
- **Фикс:** ввести `API` struct-seam (как в `SubscriptionsViewModel`), перенести `reloadPending`-паттерн; добавить тесты success/failure/guard/optimistic-revert/reload.
- **Файлы:** `SavingsViewModel.swift`, `GoalDetailViewModel.swift` + тесты.

### P1-5 · SSE 401/403 не синхронизированы с auth-политикой (iOS-F7)
- В `SSEClient` (ai-chat стрим, `:121-123`) 401 и 403 склеены в `.unauthorized`, `onUnauthenticated` не зовётся → истёкший токен в чате не разлогинит (в отличие от REST).
- **Фикс:** развести 401 (→ `onUnauthenticated` + `.unauthorized`) и 403 (`.forbidden`, без logout). Согласовать с фиксом P0-3.
- **Файлы:** `ios/.../Networking/SSEClient.swift`.

### P1-6 · ui.theme key collision (FE-F4)
- `main.tsx` shell-dispatcher (`v06`/`v10`) и `useTheme` (`maximal_poster`/`liquid_glass`/`ios_default`) пишут один `localStorage['ui.theme']` с несовместимыми словарями → выбор темы делает v06-shell недостижимым.
- **Фикс:** развести ключи (`ui.shell` vs `ui.theme`) либо удалить мёртвый v06-web-shell (см. R5).
- **Файлы:** `frontend/src/main.tsx`, `screensV10/common/useTheme.ts`.

### P1-7 · Регресс-сеть для app-wide auth/date изменений (QA-F3, QA-F4)
- `APIClient` 403-no-logout/401-always-logout и MSK `yyyy-MM-dd` decode-pin — ноль прямых тестов (хотя `session`+`onUnauthenticated` инъецируемы). Высокий риск тихой регрессии в других экранах.
- **Фикс (после P0-3):** `URLProtocol`-stub тесты: 403→logout (строгий, после P0-3); 401→always logout; decode `"2027-01-01"` через production decoder == MSK-midnight + timestamp-форматы (`createdAt`) всё ещё парсятся.
- **Файлы:** новые `APIClientForbiddenTests`, `APIClientDateDecodeTests`.

# ПЛАН 2 — Cleanup-рефакторинг (sequenced)

Порядок выбран по принципу «сначала дешёвое и снижающее риск дрейфа, потом структурное».

### R1 · iOS: устранить дублирование (низкий риск, высокий ROI)
- **Account-label** → единственный источник `AccountPickerLogic.label`; перевести `SavingsDepositSheet:65`, `TransactionEditor:228` (inline-обёртка), `SubscriptionEditor:585` (+ устранить расхождение сепаратора `" · "` vs `" ·"`). Кандидаты в FeaturesV10 тоже.
- **Mutation-error banner** — идентичный `mutationErrorBanner(_:)` в SavingsView/GoalDetailView/SubscriptionsView → общий `ViewModifier`.
- **`LocalNotifications.reschedule`** — две перегрузки (~48 строк дубля) → один generic-helper; проверить и удалить мёртвую legacy `[SubscriptionDTO]`-перегрузку.
- **`APIError → RU-копия` mapper** (`APIError.userFacingRu`) — заменить весь кластер `error.localizedDescription` (закрывает P1-3 системно).
- **Dead code:** `lastCreatedGoalId` + `clearLastCreatedGoalId()` (никто не читает) — подключить scroll-to-new или удалить.

### R2 · iOS: унификация test-seam (закрывает P1-4 структурно)
- `SavingsViewModel`/`GoalDetailViewModel` → инъецируемый `API` struct (как `SubscriptionsViewModel`). Открывает поведенческое тестирование денежных мутаций.

### R3 · iOS: схождение legacy/V10 API-слоя (ARCH-A2)
- Для каждого route выбрать canonical enum; `@available(*, deprecated)` на дублях (`ActualAPI`/`ActualV10API`, `CategoriesAPI`/`V10`/`WriteAPI`, `SubscriptionsAPI`/`V10`); мигрировать вызовы на V10 (суперсет). Завести debt-реестр из «legacy↔V10» комментариев (ARCH-A8).

### R4 · Контракт: codegen из OpenAPI (ARCH-A3) — наибольший ROI против дрейфа
- Генерировать TS (`openapi-typescript`) и Swift DTO из FastAPI OpenAPI. Сначала read-DTO (самые расходящиеся), затем request-bodies. Удалить «pending schema» Optional-заглушки (`CategoryV10`), синхронизировав `CategoryRead`. Убирает целый класс runtime-несоответствий (3 рукописных набора типов).

### R5 · Web: судьба мёртвого v06-shell (FE cleanup)
- `App.tsx` + `screens/*` + legacy `components/*` + `api/subscriptions.ts` + `hooks/useSubscriptions.ts` грузятся только при недостижимом `theme==='v06'` (~50+ файлов). Решить: вернуть достижимость или удалить (сокращение бандла + поверхности ревью). Связано с P1-6.
- `window.alert`→`Toast`; централизовать money-парсер (`parseRublesToKopecks`) и date-парсер (`parseWireDate`).

### R6 · Продуктовое решение: судьба двух iOS-шеллов (ARCH-A1) — требует владельца
- `MainShell` (v06) ↔ `V10MainShell` сейчас ROADMAP-ом зафиксированы как «постоянная альтернатива». Подтвердить: sunset одного (тогда вынести общие ViewModel/Data в shared-слой) ИЛИ официально «две темы навсегда» + инвестиция в общий доменный слой, чтобы экраны не дрейфовали (Savings/Subscriptions уже пишутся в обоих мирах).

### R7 · Cross-cutting абстракции (ARCH-A5, A6) — после стабилизации контракта
- Вынести error-policy из `APIClient`-switch в инъектируемую стратегию (фичи объявляют политику декларативно, не булевыми флагами — закрывает корень P0-3).
- Тип `BusinessDate` (DATE) отдельно от audit-времён на всех клиентах; MSK-семантика — свойство типа, не оговорка в decoder (закрывает корень MSK-pin).

### R8 · Backend-гигиена
- `est_cost_usd: Float` → `BIGINT cost_cents` (нарушает «никаких float» из CLAUDE.md).
- `MeResponse`-сборка дублируется (`router.get_me` vs `me.patch_me`) → общий билдер (предотвращает асимметрию P2-6).
- `get_db` дублируется (`dependencies.py` vs `session.py`) → один источник.
- `KindStr = ActualKindStr` alias — закончить переименование, убрать legacy-символ.
- Зафиксировать как известное ограничение: `_user_locks`/`_rate_buckets`/`_usage_buffer` per-process (cap/rate-limit/usage разъедутся при multi-worker).

### R9 · Документация (ARCH-A7)
- CLAUDE.md/HLD: система de-facto multi-tenant (RLS активен, owner/member роли работают) — обновить «single-tenant, RLS не вводим». Это актив, не carrying cost.
