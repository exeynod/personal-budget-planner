# Sketch 005: Plan / Template / Categories

## Design Question
Три родственных экрана (план месяца, шаблон, категории) — один структурный
паттерн «список + редактирование». Какая раскладка лучше: flat-список со
swipe и bottom-sheet, группировка по категории с inline-редактированием,
или единый «менеджер» с segmented control?

## How to View
```
open .planning/sketches/005-plan-and-categories/index.html
```

## Variants
- **A: Flat list + swipe + bottom-sheet** — Plain список plan-строк, swipe-actions (✎/🗑) для редактирования. Bottom sheet (как в 002-B) для добавления/правки строки. Toolbar сверху: «Шаблон / В шаблон / Фильтр». Категории и шаблон — отдельные экраны с тем же паттерном.
- **B: Grouped by category + inline edit** — Группировка по категории, каждая группа — collapsible-карточка с total суммой. Внутри — inline-input для суммы, ✕ для удаления. Кнопка «+ Добавить в Дом» прямо в группе. Best для редактирования шаблона / месяца целиком.
- **C: Segmented manager** — Один экран с segmented control сверху: «План мес. / Шаблон / Категории». Каждый pane имеет свой контент, но общий header и MainButton. Минимизирует навигацию между связанными сущностями.

## What to Look For
- **Edit-flow:** A = swipe → sheet (2 действия), B = клик в input (1), C — зависит от pane.
- **Группировка:** B сразу видит группы и тоталы, A — flat (нужен фильтр), C — зависит от выбранного pane.
- **Навигация между План/Шаблон/Категории:** A = разные экраны (нав-стек), B = тот же паттерн на трёх экранах, C = один экран.
- **Inline vs sheet:** B быстрее правит сумму, но не позволяет менять категорию/описание/день без отдельного UI. A через sheet — медленнее, но даёт полный редактор.
- **Категории-pane (только в C):** drag-handle для sort_order, toggle архивации, kind-пилюля.

## Notes
- Plan-summary банер показывает агрегаты (как и сводка дашборда, но компактно).
- В A toolbar: «↻ Шаблон» (применить шаблон заново), «⤓ В шаблон» (snapshot текущего плана в шаблон).
- В C добавлен экран категорий с drag, архивацией (toggle), kind-пилюлями (расх/дох).
- MainButton: «Добавить строку» / «Сохранить» / «+ Добавить» — меняется по контексту.

## Implementation note (gap для реализации)
Winner B покрывает только редактирование суммы. Полный редактор строки
(сменить категорию, описание, day_of_period) всё равно требует sheet —
открывается тапом на название/meta-зону строки. Используем bottom-sheet
из 002-B как универсальный полный редактор, но он не показан в этом скетче.

Для категорий (UC-6) и шаблона (UC-5) применяем тот же паттерн «группировка +
inline + sheet для полного редактирования», на отдельных экранах. Tab-bar /
segmented control между ними не используется — навигация через TG BackButton.
