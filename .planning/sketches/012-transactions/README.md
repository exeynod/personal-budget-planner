# Sketch 012 — Транзакции (top-level tab)

## Design Question
Как выглядит таб «Транзакции» — объединение `History` + `Plan` в одном табе под-табами.

## Variants
- **A:** Под-таб «История» активен — список факт-транзакций сгруппирован по датам, с фильтр-чипами (Все / Расходы / Доходы / По категории)
- **B:** Под-таб «План» активен — плановые строки сгруппированы по категориям, с источник-маркером (template/manual/subscription)
- **C:** Empty-состояние «История»

## Background
До v0.3 были два разных экрана: `HistoryScreen` (из таба «История») и `PlannedScreen` (из таба «План»). В новой функциональной nav оба сворачиваются в один таб «Транзакции». Под-табы переключают режим: факт vs план.

## Style
- Top-level: PageTitle с h1 без back-button
- Под-табы: underline TabBar (как Расходы/Доходы на Главной)
- Фильтр-чипы: pill-стиль на surface, активный = primary-soft
- История: группировка по дням с total за день (uppercase muted day-header)
- План: группировка по категориям, источник как badge

## Winner
**All valid ★** (подтверждено 2026-05-05) — три варианта представляют разные состояния одного экрана, все три нужно реализовать в Phase 7:
- A → нормальный режим под-таба «История» с заполненными днями
- B → нормальный режим под-таба «План» с группировкой по категориям
- C → empty-state когда в текущем периоде нет факт-транзакций
