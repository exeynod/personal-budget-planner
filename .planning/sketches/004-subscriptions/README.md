# Sketch 004: Subscriptions

## Design Question
UC-7 (управление подписками) + UC-8 (push-напоминания о списаниях). Главный
вопрос дизайна — как показать «когда» — это критичный для пользователя
параметр. Какой паттерн быстрее отвечает на «что мне списывают на этой
неделе»?

## How to View
```
open .planning/sketches/004-subscriptions/index.html
```

## Variants
- **A: Список + горизонтальный таймлайн** — Hero с суммой за период и ближайшим списанием. Таймлайн-карточка: горизонтальная шкала «5 фев → 4 мар» с точками-списаниями, today-line, цветовая логика (красный = ≤2 дня, жёлтый = ≤7, нейтральный = дальше). Под ней список карточек с pill-индикатором «через N дней».
- **B: Календарная сетка** — Hero, ниже календарь месяца 7×6 со стандартной разметкой. Точки на днях списаний (warn/danger). Лист подписок ниже — без сложной сортировки.
- **C: Секции по срочности** — Hero, ниже четыре секции: «⚠ Ближайшие 3 дня», «На этой неделе», «До конца месяца», «Позже». Без визуализации временной шкалы. Каждая секция показывает суммарный итог.

## What to Look For
- **Скорость ответа «что списывается завтра»**: A (точка с pulse), B (день в календаре), C (первая секция).
- **Ощущение «нагрузки»**: A показывает кластеры списаний на одной шкале, B даёт календарную карту, C просто список.
- **Понятность**: таймлайн (A) требует объяснения, календарь (B) знаком всем, секции (C) самые понятные.
- **Density**: A и C компактнее, B забирает много вертикали под календарь.
- **Расширяемость**: что лучше масштабируется на 30+ подписок и на горизонт 3 месяца.

## Notes
- Hero отображает «Сумма за период» (6 949 ₽ — суммарная нагрузка на этот период) + ближайшее списание / счётчик активных.
- Cycle-бейдж: «мес» / «год» рядом с названием.
- Урочные карточки получают цветной border-left (warn/danger).
- MainButton — «Добавить подписку».
