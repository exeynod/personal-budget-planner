# Screen Inventory

Полный список экранов с описанием, состояниями и интеракциями.
Привязка к коду — `prototype/poster-screens.jsx`.

---

## 00. Onboarding

**Файл:** `PosterOnboarding`, шаги `OnbStep1Income` → `OnbStep4Goal` + `OnbDone`.

**4 шага:**

### 01 · Доход

- Hero: «Какой доход в месяц?»
- Eyebrow: «ВВЕДИ СУММУ ПОСЛЕ НАЛОГОВ»
- Большой инпут с символом ₽ справа, валидация числа.
- NEXT disabled пока сумма не > 0.

### 02 · Счета

- Hero: «Откуда деньги?»
- Список карточек-чипов: Т-Банк / Сбер / Наличные / + Добавить.
- Можно выбрать несколько; первая выбранная = primary.
- Список выбранных с возможностью указать баланс.

### 03 · План

- Hero: «Распредели **{income} ₽**»
- Eyebrow: «СДВИГАЙ ПОЛЗУНКИ ПО КАТЕГОРИЯМ»
- 8 дефолтных категорий с слайдерами (initial — пропорция от дохода).
- Внизу: «остаётся X ₽ → накопления» / «превышение на X ₽».
- NEXT disabled при превышении.

### 04 · Цель (опц.)

- Hero: «Зачем копишь?»
- Eyebrow: «МОЖНО ПРОПУСТИТЬ И НАСТРОИТЬ ПОЗЖЕ»
- 2 поля: название цели, сумма.
- Кнопка «ПРОПУСТИТЬ» сверху (skip к финалу).

### Final · ВСЁ

- Eyebrow: «VOL.04 · ГОТОВО»
- Mass UPPERCASE: «ВСЁ.»
- Mass italic: «деньги — под контролем.»
- Резюме: ДОХОД / СЧЕТА / ПЛАН / ЦЕЛЬ.
- CTA: «НАЧАТЬ →».

**Хром (OnbChrome):** прогресс-полоска (4 деления), слева `←`, по центру eyebrow
«ШАГ 0X / 04», справа «ПРОПУСТИТЬ» (если применимо).

## 03. AI

**Файл:** `PosterAi`. Фон: cream, текст ink.

**Initial state (без сообщений):**

- Eyebrow: «AI · ASSISTANT / ONLINE»
- Большая фраза-наблюдение DM Serif 36px («Май в плюсе на **21 170 ₽**.»)
- Под ней дополнение DM Serif italic 24px («Кафе уже **+23%** к лимиту — стоит притормозить.»)
- Eyebrow: «— из ваших данных, 9 мая»
- Eyebrow: «ПОДСКАЗКИ · ТАПНИ» + 4 строки-чипа (DM Serif italic 18px) с `→`.

**Active state (есть сообщения):**

- Лента: user — чёрная плашка справа, ai — Italic-текст слева в рамке.
- Индикатор «печатает» — три точки.
- Авто-скролл к последнему.

**Инпут (всегда):**

- Чёрная плашка снизу, моно-инпут, плейсхолдер «напишите или тапните подсказку…»
- Жёлтая кнопка «↵ ОТПРАВИТЬ».

## 08. Счета

**Файл:** `PosterAccounts` + `PosterAccountDetail`. Фон: cream / black.

### Список

- Mass italic «Счета.»
- Тёмная плашка «СУММАРНО · X ₽ · N счетов».
- Список счетов: bank · type/mask · balance · «история →».
- Бейдж «ОСНОВНОЙ» у primary.
- CTA внизу: «+ ДОБАВИТЬ СЧЁТ» / «ПЕРЕВОД».

### Детальный счёт

- Mass italic «Т-БАНК.»
- Подпись: «карта ·· 4408» / «наличные» / «накопит. счёт».
- 2 KPI: «БАЛАНС» (жёлтый) + «В МАЕ · N ОПЕРАЦ.» (тёмный).
- История операций по этому счёту, сгруппированная по дням.

**Empty state:** «По счёту пока нет операций.» (R5).

## 10. Add Sheet (новая операция)

```

кафе / продукты / …
─────────────────────────

[СЕГОДНЯ] [ВЧЕРА] [СВОЯ ДАТА]

[КАФЕ] [ПРОДУКТЫ] [ДОМ] [ТРАНСПОРТ] ...

[1] [2] [3]
[4] [5] [6]
[7] [8] [9]
[.] [0] [⌫]

[СОХРАНИТЬ ↵]
```

**Состояния CTA:**

- amount = 0 → «ВВЕДИТЕ СУММУ» (disabled).
- amount > 0, нет категории → «ВЫБЕРИТЕ КАТЕГОРИЮ» (disabled).
- amount > 0 + категория → «СОХРАНИТЬ ↵» (active, yellow).

**Закрытие:**

- Если есть несохранённые данные — confirm-sheet:
  - «ОТМЕНИТЬ ЗАПИСЬ?»
  - «Сумма X ₽ не сохранена.»
  - [ПРОДОЛЖИТЬ] [ОТМЕНИТЬ].

```

       ├──→ TXN (Все операции →)
       ├──→ BUDGET (PLAN бейдж)
       ├──→ ACCOUNTS (в кошельке →)
       └──→ MGMT (МЕНЮ ↗)

MGMT ──┬──→ BUDGET
       ├──→ ACCOUNTS ──→ ACCOUNT DETAIL
       └──→ ANALYTICS

BUDGET ──→ SUBS (ВСЕ ПОДПИСКИ →)
BUDGET ──→ CATEGORY (тап по строке категории)

TAB BAR ──┬──→ HOME
          ├──→ SAVINGS
          ├──→ AI
          └──→ MGMT

FAB ──→ ADD SHEET (поверх любого экрана)
```
