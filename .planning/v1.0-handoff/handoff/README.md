# TG Budget — Maximal Poster

Пакет для передачи в разработку. Внутри:

```
handoff/
├── README.md            ← вы здесь
├── ТЗ.md                ← полное техзадание (читать первым)
├── DESIGN-SYSTEM.md     ← цвета, шрифты, токены, компоненты
├── SCREENS.md           ← инвентарь экранов с описанием каждого
├── DATA-MODEL.md        ← структуры данных, транзакции, бизнес-правила
└── prototype/           ← рабочий HTML-прототип (открыть index.html)
    ├── index.html
    ├── poster-app.jsx        — навигационная оболочка, iOS-фрейм, sheet, тостеры
    ├── poster-screens.jsx    — все экраны (≈1500 строк)
    └── tweaks-panel.jsx      — служебная панель (можно не переносить в продакшн)
```

## Как читать

1. **ТЗ.md** — общее видение, флоу, бизнес-правила, требования к каждому экрану.
2. **DESIGN-SYSTEM.md** — палитра, типографика, отступы, состояния — то, что должно стать токенами.
3. **SCREENS.md** — пошаговое описание каждого экрана: что на нём, какие интеракции, edge-cases.
4. **DATA-MODEL.md** — формат сущностей (категория, счёт, транзакция, цель), форматтеры, правила переносов остатков.
5. **prototype/** — открыть `index.html` локально (нужен http-сервер из-за `fetch()` для JSX-модулей; подойдёт `python -m http.server` или Live Server в VS Code).

## Технологический стек прототипа

Прототип сделан на React 18 + Babel inline (трансформация JSX в браузере). Для продакшна
ожидается:

- **Платформа:** Telegram Mini App (TWA) → React + Vite + TypeScript.
- **State:** локальный (Zustand / Redux Toolkit), синк с бэкендом по REST/GraphQL.
- **Стили:** CSS Modules или vanilla-extract; токены из `DESIGN-SYSTEM.md`.
- **Шрифты:** Google Fonts (см. ниже) либо self-hosted.
- **Анимации:** Framer Motion или CSS-анимации (в прототипе — CSS keyframes).

## Шрифты

```
Archivo Black            (заголовки, eyebrow, кнопки)
DM Serif Display Italic  (акцентные фразы, AI-наблюдения, заголовки экранов)
JetBrains Mono           (цифры, моноширинные ярлыки)
Manrope 400/500/600/700  (основной текст)
```

Все четыре — Google Fonts, бесплатные.

## Что НЕ нужно переносить в продакшн

- `tweaks-panel.jsx` — служебная панель для дизайн-итераций.
- Левая/правая панели в `poster-app.jsx` (быстрые переходы, индикатор «сейчас») — это
  оболочка для демо в браузере.
- Хардкод данных в `poster-screens.jsx` (`POSTER_DATA`) — заменить на запросы к API.

## Что важно сохранить 1:1

- **Цветовая палитра** и применение цвета к экранам.
- **Типографические контрасты:** мелкий моно-eyebrow → огромный display → серый Italic.
- **Анимации появления** строк (stagger 45ms), счётчики (count-up), bar-fill, tab-pop.
- **Поведение «остаток категории»** (переключатель → накопления / прочее).
- **Округление трат → копилка**.
