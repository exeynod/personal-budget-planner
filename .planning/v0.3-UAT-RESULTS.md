### Steps (повтор из v0.3-UAT-PLAN.md, для удобства)

Запусти Mini App в браузере: <http://localhost/> (Caddy отдаёт SPA).

1. **UAT-1 — SSE streaming визуал** (Phase 9)
   Таб «AI» → клик suggestion-chip «Каков мой баланс?» → текст должен
   появляться **постепенно**, не одним блоком.
2. **UAT-2 — Suggestion chips в empty-state**
   Корзина → удалить историю → 4 chip'а («Каков мой баланс?», «Где я
   перерасходовал?», «Сколько потратил на еду?», «Сделай прогноз»)
   видны над input bar; клик отправляет запрос, chip'ы исчезают.
3. **UAT-5 — AI suggestion box появляется** (Phase 10)
   Транзакции → FAB → ввести «продукты» (≥3 chars) → через ~500ms
   AI-suggestion box с категорией «Продукты» и confidence-bar.
4. **UAT-6 — Confidence bar render**
   Тот же экран — ширина бара пропорциональна `confidence`,
   фиолетовый `#a78bfa`, без overflow.
5. **UAT-7 — Кнопка «Сменить»**
   В suggestion box нажать «Сменить» → AI-box исчезает, появляется
   стандартный `<select>` со всеми категориями.
6. **UAT-8 — Settings toggle**
   Управление → Настройки → выключить «AI категоризация» → в
   ActualEditor suggestion box не появляется. Включить → снова работает.

**Тестовые запросы для UAT-5/6** (которые гарантированно дают
suggestion ≥0.5 на текущих 9 категориях):

- «продукты пятёрочка» → Продукты (~0.59)
- «Кафе и рестораны» → Кафе и рестораны (~1.0, exact match)
- «Подарки» → Подарки и сюрпризы (~1.0)

Замечание про threshold: `SUGGEST_THRESHOLD=0.5` — это довольно жёсткий
порог для `text-embedding-3-small`. Многие синонимы («тренировка»→Спорт,
«такси»→Транспорт) не дотянут. Это **design-выбор Phase 10**, не баг.
Если в проде окажется недотюнено — кандидат на снижение до ~0.4 в
follow-up (out of scope этого аудита).

## Bugs found & fixed during UAT

В процессе авто-прогона нашлось 3 реальных бага в коде (1 в Phase 10.1
как новый, 2 pre-existing из Phase 10 — никогда не сработали раньше,
т.к. `OPENAI_API_KEY` не пробрасывался в api контейнер). Все
исправлены в одном коммите перед публикацией результатов:

| # | Где | Симптом | Фикс |
|---|-----|---------|------|
| 1 | `app/api/routes/categories.py::create_category` / `update_category` | `ForeignKeyViolationError: category_id not present in table "category"` в background-task `_refresh_embedding` | Явный `await db.commit()` перед `background_tasks.add_task(...)` — request-сессия должна закоммититься до того, как новая сессия фоновой задачи попытается прочитать row |
| 2 | `app/api/routes/categories.py::_refresh_embedding` | INSERT в `category_embedding` уходил, лог `category.embedding.refreshed` писался, но **в БД ничего не появлялось** (Phase 10 latent bug) | `async with AsyncSessionLocal() as session:` не auto-commit'ит. Добавлен явный `await session.commit()` |
| 3 | `app/ai/embedding_service.py::suggest_category` | `asyncpg.DataError: invalid input ... expected str, got list` (Phase 10 latent — pgvector raw-SQL cast не принимает Python list через text-param) | Преобразование `list[float]` → строка-литерал `'[0.1,0.2,...]'` перед подачей в `:query_vec` |
| 4 | `app/ai/embedding_service.py::get_embedding_service` | LRU-кэш не работал — каждый запрос получал новый instance с пустым `_embed_cache` | `@lru_cache(maxsize=1)` на factory → singleton |

(1) — это новый bug Phase 10.1 (T-4.1 не работал бы в проде). (2)+(3) —
pre-existing Phase 10 баги, но обнаружены только сейчас потому что
до Phase 10.1 ключ OpenAI не пробрасывался в api контейнер
(см. `79edf38 fix(infra)`), и код пути embedding refresh / suggest
никогда не выполнялся в живой среде.
