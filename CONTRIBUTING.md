# Contributing

Thanks for considering contributing to TG Budget Planner open-core.

## What's open для contributions

- **Open-core components** только. См. `OPEN-CORE-MANIFEST.md`.
- AI/iOS/Maximal Poster components — proprietary, не accept PRs.

## How to contribute

1. **Issue first** — для нового feature или нон-trivial bugfix создай issue + дискуссия.
2. **Fork + branch** — `feat/short-name` или `fix/short-name`.
3. **Tests** — для backend новые тесты обязательны (pytest).
4. **Style** — backend: ruff format/check; frontend: prettier.
5. **Commit message** — Conventional Commits prefix (feat/fix/docs/refactor/test/chore).
6. **PR** — описание + screenshots если UI + checklist.

## DCO (Developer Certificate of Origin)

Все commits должны быть подписаны через `git commit -s` подтверждая DCO.

## Code of Conduct

Be respectful. Personal attacks → ban. Question implementations, not authors.

## Local dev setup

См. `docs/dev-setup.md`.
