# ADR-0003: Общий `app/` + три точки входа (api/bot/worker, один образ)

- **Статус**: Accepted
- **Дата**: 2026-06-07
- **Контекст**: `deploy/Dockerfile` (build-arg `SERVICE`), `main_api.py` / `main_bot.py`
  / `main_worker.py`, `deploy/docker-compose.yml`, `app/core/settings.py::SERVICE`

## Контекст и проблема

Три рантайма (FastAPI, aiogram-бот, APScheduler-воркер) делят один домен:
модели, сервисы, period-движок, RLS-scope, AI-tools, настройки. Дублировать эту
логику по трём кодовым базам — гарантированный дрейф (модель/правило поменяли в
одном месте, забыли в двух других).

Три отдельных Docker-образа на одном VPS — это тройная сборка зависимостей и
холодный кэш слоёв при каждом изменении общего кода.

При этом точки входа и наборы обязательных секретов у сервисов разные (AI-ключ
нужен только api), и это надо как-то развести.

## Рассмотренные варианты

1. **Три отдельных проекта/образа (status quo)** — изоляция, но дублирование
   домена и тройная сборка. Отклонено.
2. **Монолит-процесс (api+bot+worker в одном)** — нет дублирования, но смешение
   жизненных циклов (long-poll, cron, HTTP) в одном процессе хрупко и мешает
   масштабировать/рестартить по отдельности. Отклонено.
3. **Общий `app/` + три тонкие точки входа, один образ, выбор по env (выбран).**

## Решение

Весь домен — в `app/` (модели, сервисы, core, ai, db). Три тонких файла
`main_api.py` / `main_bot.py` / `main_worker.py` — только wiring точки входа.
Один `deploy/Dockerfile` (build-arg + env `SERVICE`); `CMD` ветвится по `SERVICE`
(`api`→`deploy/entrypoint.sh`, `bot`→`main_bot.py`, `worker`→`main_worker.py`).
Слои deps + `app/` кэшируются и переиспользуются всеми тремя сервисами.
`validate_production_settings` смотрит на `SERVICE`, чтобы требовать AI-секрет
только у api.

## Последствия

### Хорошие

- Нет дублирования домена — модель/правило меняется в одном месте.
- Тёплый кэш слоёв: общий код пересобирается один раз на все три сервиса.

### Плохие

- Образ тащит зависимости всех трёх рантаймов (aiogram в api-контейнере и т.п.) —
  чуть жирнее.
- Связанность: ломающее изменение в `app/` затрагивает все три сервиса разом.

### Нейтральные

- Env `SERVICE` обязателен — определяет и точку входа, и набор проверок секретов.
