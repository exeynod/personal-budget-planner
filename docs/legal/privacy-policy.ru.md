# Политика обработки персональных данных

**Версия:** v0.1 (2026-05-11)
**Применимое право:** Федеральный закон РФ от 27.07.2006 № 152-ФЗ
«О персональных данных».

## 1. Оператор персональных данных

Оператор — частное физическое лицо, владелец сервиса TG Budget Planner.
Контакт: **exeynod@gmail.com**.

## 2. Цели обработки персональных данных

Персональные данные (далее — ПДн) обрабатываются исключительно для:

1. Аутентификации пользователя через Telegram Mini App (Telegram initData).
2. Хранения бюджетных данных пользователя (категории, транзакции, планы,
   подписки) для отображения в приложении и боте.
3. Отправки push-уведомлений через Telegram бот (только subscribed user'ам).
4. Аналитики работы AI-помощника (тратa токенов, ai_usage_log) для
   контроля себестоимости.
5. Audit-журнала операций над ПДн (consent / экспорт / удаление) для
   compliance с §10.1 152-ФЗ.

ПДн **не используются** для маркетинга, профилирования третьими
лицами или передачи рекламодателям.

## 3. Виды обрабатываемых ПДн

| Категория | Источник | Назначение |
|-----------|----------|------------|
| Telegram user ID (tg_user_id) | Telegram initData | Идентификация |
| Telegram chat ID (tg_chat_id) | /start команда | Push-уведомления |
| Имя/username в Telegram (опционально) | Telegram initData | UI display |
| Бюджетные данные (категории, транзакции, планы, накопления, цели) | Пользователь | Функционал приложения |
| Текст сообщений AI-чату | Пользователь | Обработка через OpenAI |
| IP-адрес (хэш) | HTTP request | Audit log (sha256) |

## 4. Правовое основание обработки

Основанием обработки является **согласие субъекта ПДн** (ст. 6 ч. 1 п. 1
152-ФЗ), полученное явно через onboarding-чекбокс «Я согласен на
обработку персональных данных» с предварительным ознакомлением
с настоящей Политикой. Согласие может быть отозвано в любой момент
через `DELETE /api/v1/me/consent` или удаление аккаунта.

## 5. Сроки хранения ПДн

- Активный пользователь: ПДн хранятся в течение всего срока использования
  сервиса.
- После запроса на удаление аккаунта: данные помечаются soft-delete
  (`app_user.deleted_at`) и удаляются окончательно через **30 дней**
  (cooling period для отмены случайного удаления).
- После окончательного удаления: audit-записи (с захэшированным user_id)
  хранятся ещё **1 год** для compliance с регуляторными запросами,
  затем архивируются.

## 6. Способы обработки

Обработка осуществляется автоматизированным способом с использованием
средств вычислительной техники. Никакая ручная обработка ПДн оператором
не предусмотрена.

## 7. Хранение

ПДн хранятся на серверах в РФ (VPS оператора). Резервные копии
периодически снимаются через `pg_dump` и хранятся в зашифрованном виде.

## 8. Передача ПДн третьим лицам (sub-processors)

| Sub-processor | Цель | Данные | Расположение серверов |
|---------------|------|--------|------------------------|
| **OpenAI** | AI-чат / категоризация | Текст сообщений пользователя в AI-чате; названия категорий | EU (api.openai.com EU residency) |
| **Telegram** | Доставка сообщений / Mini App | tg_user_id, текст бот-сообщений | Зависит от инфраструктуры Telegram |

Никаким другим третьим лицам ПДн не передаются.

## 9. Права субъекта персональных данных

Согласно ст. 14-15 152-ФЗ, субъект ПДн имеет право на:

1. **Доступ к ПДн** — через `GET /api/v1/me/export` (JSON-выгрузка всех
   данных пользователя).
2. **Исправление неточных ПДн** — через UI приложения (любые поля
   редактируются).
3. **Удаление ПДн** («право на забвение») — через `DELETE /api/v1/me/account`
   (soft-delete + автоматический hard-delete через 30 дней).
4. **Отзыв согласия** — через `DELETE /api/v1/me/consent` (после отзыва
   приложение блокирует все основные операции до повторного согласия).
5. **Обжалование действий оператора** — обращение к оператору через
   email **exeynod@gmail.com** или в Роскомнадзор.

## 10. Защита ПДн

1. Все API-запросы проходят через HTTPS (TLS via Let's Encrypt).
2. PostgreSQL Row-Level Security (RLS) обеспечивает изоляцию данных
   пользователей: каждый запрос выполняется под non-superuser ролью
   с принудительной фильтрацией по `app.current_user_id` GUC.
3. Telegram initData валидируется HMAC-SHA256 на каждом запросе.
4. Audit-журнал хранит только захэшированные (sha256) user_id и IP —
   идентификация невозможна после удаления аккаунта.

## 11. Cookies

Приложение использует только **технически обязательные** cookies для
функционирования Mini App (Telegram session). Аналитические /
маркетинговые cookies не используются.

изменения сопровождаются повторным запросом согласия в onboarding-flow.
