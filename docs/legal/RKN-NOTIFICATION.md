# Шаблон уведомления Роскомнадзора об обработке ПДн

> **Manual user-side action.** Подача уведомления выполняется владельцем
> сервиса через онлайн-форму на [pd.rkn.gov.ru](https://pd.rkn.gov.ru/operators-registry/notification/form/).
> Требует ЭЦП или аутентификации через Госуслуги. Автоматизация
> невозможна без user-side credentials.

## Шаги подачи

1. Открыть https://pd.rkn.gov.ru/operators-registry/notification/form/
2. Авторизоваться (Госуслуги / ЭЦП).
3. Заполнить поля формы значениями из секции «Содержание уведомления» ниже.
4. Submit. После проверки РКН (до 30 дней) — выдаст reg-номер оператора.
5. Записать reg-номер в `docs/COMPLIANCE.md` (поле `rkn_registration_id`).

### Цели обработки персональных данных

1. Аутентификация пользователя через Telegram Mini App для предоставления
   доступа к функционалу ведения личного бюджета.
2. Хранение бюджетных данных пользователя (категории расходов и доходов,
   транзакции, плановые показатели, подписки на регулярные платежи)
   для отображения в приложении и Telegram-боте.
3. Отправка push-уведомлений через Telegram-бот пользователю
   (только подписавшимся).
4. Аналитика работы AI-помощника (потребление токенов) для контроля
   себестоимости сервиса.
5. Ведение audit-журнала операций над персональными данными (получение
   согласия, экспорт, удаление) в соответствии со ст. 18.1 152-ФЗ.

### Правовое основание обработки

Согласие субъекта персональных данных, полученное в письменной форме
(через checkbox-механизм в Telegram Mini App) после ознакомления с
Политикой конфиденциальности (опубликована на `/legal/privacy`).
Ст. 6 ч. 1 п. 1 152-ФЗ.

### Описание категорий субъектов ПДн

Физические лица — пользователи сервиса TG Budget Planner, добровольно
подписавшие согласие на обработку своих ПДн.

### Перечень обрабатываемых ПДн

1. Идентификатор пользователя Telegram (tg_user_id) — численный.
2. Идентификатор чата Telegram (tg_chat_id) — численный.
3. Имя и/или username в Telegram (если указаны пользователем) — текстовый.
4. Бюджетные данные: суммы транзакций, названия категорий, плановые
   показатели, накопления, цели, подписки — числовые и текстовые.
5. Текст сообщений пользователя AI-чату — текстовый.
6. Хеш IP-адреса (sha256) — для audit-журнала.

### Способы обработки

Автоматизированная обработка с использованием средств вычислительной
техники. Ручная обработка ПДн оператором не осуществляется.

### Сроки хранения

- В период активного использования сервиса: бессрочно (до запроса
  на удаление).
- После запроса на удаление: 30 дней soft-delete + полное физическое
  удаление через автоматизированную задачу.
- Audit-журнал (с захэшированным user_id): 1 год после удаления
  аккаунта, затем архивация.

### Меры защиты ПДн

1. **Технические:** TLS-шифрование (Let's Encrypt) на all API-запросах,
   PostgreSQL Row-Level Security с принудительной фильтрацией по
   идентификатору пользователя, HMAC-SHA256 валидация Telegram initData
   на каждом запросе, аудит-журнал с захэшированными идентификаторами.
2. **Организационные:** доступ к серверу — только у владельца оператора;
   backup'ы базы данных шифруются перед хранением.

### Лица, осуществляющие обработку

Обработку осуществляет лично владелец сервиса (оператор) с использованием
автоматизированных систем.

## После получения reg-номера

1. Зайти в `docs/COMPLIANCE.md`.
2. Заполнить поля:
   - `rkn_registration_id: <выданный РКН номер>`
   - `rkn_notified_at: <дата получения reg-номера>`
3. Закоммитить с message `docs(compliance): record РКН registration <id>`.
4. Закрыть REQ-33-01 в `.planning/REQUIREMENTS.md` (поставить `[x]`).
