## WI-A (делается сейчас): глобальные exception-handlers

Root `main_api.py` (FastAPI). Добавить:

1. `RequestValidationError` → консистентный 422 (сохранить текущую форму, если фронт на неё завязан).
2. Необработанный `Exception` → 500 чистым JSON (без утечки стека наружу) + лог трейсбека.
3. Доменный `ValueError` (напр. `CategoryKind(...)` в PATCH /actual roundup/deposit) → 4xx вместо 500.
   Это попутно чинит handoff-баг «PATCH /actual roundup/deposit → 500».

- НЕ менять 103 существующих `raise HTTPException` (они уже отдают корректные коды).
- Тест: необработанный ValueError из сервиса → чистый 4xx/500 (не голый traceback).
- Доки: `docs/services/api.md` (раздел про error-handling) в том же изменении.
